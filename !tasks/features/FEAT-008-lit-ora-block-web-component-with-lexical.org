* Initial Message

* Intention & Purpose
Build the per-block Lit web component =<ora-block>= that hosts one Lexical editor instance, plus the LiveView hook =BlockEditor= that wires it to the server: lock acquire on focus, lock release on blur, debounced =:update_content= save, render-from-server on remote update. Render =:paragraph=, =:heading_1/2/3= types end-to-end in =ConceptWeb.PageLive=. Read-only render for blocks held by other users.

* Context & Narrative
This is the first integration of Lit + Lexical + LV + Ash. We restrict to the 4 types from FEAT-006 to prove the loop; remaining types in FEAT-010/012 plug in via the BlockType registry without rewiring this layer.

* Scope
** In-scope
- =assets/js/components/ora-block.js= Lit element:
  - Attributes: =block-id=, =block-type=, =initial-content= (JSON-encoded EditorState), =read-only= (bool), =placeholder= (string)
  - Mounts a Lexical headless editor in =firstUpdated()= using =createEditor({ namespace: "ora", nodes: [...registry], theme: ora.theme })=
  - Loads initial state via =editor.setEditorState(editor.parseEditorState(JSON))=
  - Listens to editor updates → emits ={detail: {state}}= CustomEvent =ora-block-change= (debounced 400ms)
  - Emits =ora-block-focus=, =ora-block-blur=, =ora-block-arrow-up=, =ora-block-arrow-down=, =ora-block-enter-at-end=, =ora-block-backspace-at-start=
  - =read-only= sets =editor.setEditable(false)=
- =assets/js/lexical/registry.js=: imports the four lexical core nodes (=ParagraphNode=, =HeadingNode=, =TextNode=, =LineBreakNode=) plus =@lexical/rich-text= bootstrapping. Mapping from BlockType =lexical_node= string to JS class.
- =assets/js/hooks/block_editor.js= Phoenix LV Hook attached to =<ora-block>= host:
  - =mounted()=: listens to ora-block-* events; pushes server events =:focus_block=, =:blur_block=, =:save_content=, =:nav_block=
  - =updated()=: when server pushes new initial-content (e.g. another user edited), calls =element.applyRemote(state)= which uses =editor.setEditorState= without firing local change handler (guard flag)
  - Lock handling: on focus → push =:focus_block= → server attempts =acquire_lock!/2= → on success, server replies via =push_event "lock_granted"=; on failure (=:lock_held_by_other=), Hook sets =read-only=
  - Heartbeat: while focused, sends =:refresh_lock= every 15s
- =ConceptWeb.PageLive= module rendered inside =WorkspaceLive='s =:page= live_action slot:
  - =mount/3= loads page + blocks via =Concept.Pages.list_blocks_for_page!=
  - =render/1= iterates =@blocks= and renders =<ora-block>= per row with =phx-hook="BlockEditor"=
  - =handle_event "focus_block"=: =Concept.Pages.acquire_lock!/2=; on success broadcast + push lock_granted; on failure push =:lock_denied=
  - =handle_event "save_content"=: validates lock, =Concept.Pages.update_content!/2=
  - =handle_event "blur_block"=: =Concept.Pages.release_lock!/2=
  - =handle_info {:block_updated, block}=: when sender is another user, push =:apply_remote= to that block
- Static read-only render fallback (server-rendered HTML inside =<ora-block read-only>=) for first paint / non-JS / SEO

* Edge Cases
- Initial editor state JSON malformed → Lit element catches exception, falls back to empty paragraph and pushes =:client_state_recovered= event with original JSON for server-side log.
- Lock granted by server then user navigates away within 100ms: hook's =destroyed()= sends =:blur_block=; server releases.
- LV reconnect: hook =reconnected()= clears debounce + re-subscribes; if local edits pending, immediate save flush.
- Lexical's =onError= callback escalates to LV via =:editor_error= push; server logs and keeps block read-only until next focus.
- Conflicting remote update arriving while user holds lock: should not happen (lock prevents it); if it does (clock skew), discard remote, log warning.
- Caret loss on remote applyRemote: the spec says we only call applyRemote when local user does NOT hold the lock. Lexical's =setEditorState= preserves selection where possible.
