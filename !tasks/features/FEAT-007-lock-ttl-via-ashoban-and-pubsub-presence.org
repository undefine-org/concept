* Intention & Purpose
Make block locks self-healing: if a holder disconnects without releasing, an AshOban trigger releases the lock when =lock_expires_at < now()=. Plus stand up =ConceptWeb.Presence= so each =PageLive= tracks which users are viewing/editing a page (used by FEAT-015 for avatars and lock UI hints).

* Context & Narrative
Settled: AshOban trigger replaces the originally planned custom GenServer. Phoenix.Presence is a thin native fit for "who's looking at this page".

* Scope
** In-scope
- AshOban trigger on =Concept.Pages.Block=:
  - =trigger :release_expired_locks= action =:release_lock=
  - filter =lock_state == :locked and lock_expires_at < now()=
  - scheduler runs every 10s on the =:locks= Oban queue
  - trigger uses =:system= actor to bypass actor-based policies
- Configure cron-style schedule via =AshOban.Plugin= in =config :concept, Oban=
- =ConceptWeb.Presence= module using =Phoenix.Presence= against topic =workspace:#{ws}:page:#{page}:presence=
- Helper =ConceptWeb.PageChannel.track(socket, user)= called from =PageLive.mount/3= via =on_mount= hook
- Refresh action wired: when user edits a block, =ConceptWeb.PageLive= calls =refresh_lock!/2= every 15s while focus is held (heartbeat from Lit hook in FEAT-008)
- Tests covering lock-expiry trigger and presence join/leave

** Out-of-scope
- UI rendering of presence avatars (FEAT-015)
- Lock UI affordances on blocks held by others (FEAT-015)

* Existing Patterns
- AshOban: =triggers do; trigger :name do; action :acn; where expr(...); scheduler_cron "*/10 * * * * *"; end; end=
- Phoenix.Presence: =use Phoenix.Presence, otp_app: :concept, pubsub_server: Concept.PubSub=

* Tests
File: =test/concept/pages/lock_expiry_test.exs=, =test/concept_web/presence_test.exs=

Scenarios:
1. Block locked with =lock_expires_at= 1s in past; running =Concept.Pages.Block.AshOban.run_trigger(:release_expired_locks)= releases it; state =:unlocked=, holder cleared.
2. Lock with future expiry not affected by trigger.
3. =:refresh_lock= called by holder extends =lock_expires_at= by 30s.
4. Two LV processes joining same page topic appear in =Presence.list/1=; on terminate they leave.
5. PubSub: presence diff broadcasts on join/leave.

* Edge Cases
- Clock skew: server clock authoritative; client never sends timestamps.
- Trigger queue backed up: with =max_attempts: 3=, rare; document in module doc.
- Lock holder reconnects after expiry: their next =:update_content= fails with =:not_lock_holder=; LV catches and shows "Re-acquire?" toast.
- Presence flapping (5s reconnect): TTL stays 30s so brief disconnects don't release.
- Test isolation: AshOban trigger must be invoked manually, never via cron, in tests.
