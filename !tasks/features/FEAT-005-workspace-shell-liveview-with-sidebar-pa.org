* Initial Message

implement a pixel-perfect version of Notion. Use Ash as much as possible. This plan should cover the MVP version.

* Intention & Purpose
Render the Notion-style application shell: left sidebar (240px, =#F7F6F3=) with workspace header, search input (inert in this FEAT — wired in FEAT-014), nested collapsible page tree, "+ New page" affordance; main canvas area showing either an empty state or the active page slot (filled by FEAT-008+ once blocks land). Sidebar reflects =Concept.Pages= changes via PubSub.

* Context & Narrative
Shell is the persistent home for everything. We use Petal Sidebar/Dropdown/Modal for chrome, custom layout CSS for proportions. Sidebar tree expands/collapses children lazily (toggle ▶/▼ next to each row).

* Scope
** In-scope
- =ConceptWeb.WorkspaceLive= LiveView at =/w/:workspace_slug= and =/w/:workspace_slug/p/:page_id=
- =live_session :authenticated, on_mount: {LiveUserAuth, :live_user_required}=
- Sidebar layout: workspace header (icon + name + chevron-down placeholder for switcher), search input (stub), pinned actions ("New page"), page tree, footer (settings/sign-out via Petal Dropdown)
- Page tree component =ConceptWeb.PageTree= (live_component): renders nested =<ul>= with disclosure triangles, hover affordances (+/...), drag handles (Sortable hook attached but actual reorder wired in FEAT-009)
- Right-click / Petal Dropdown on row: Rename, Duplicate (FUP — disabled in MVP), Move to (FUP), Archive
- "+" affordance on hover row → creates child page via =Concept.Pages.create_page=, navigates to it
- Inline rename: double-click title → contenteditable input → Enter commits via =:rename=
- Empty state on canvas when no page selected: "Pick a page or create one" + Petal button
- PubSub subscription to =workspace:#{ws.id}:pages=; handler updates assigns and re-renders

** Out-of-scope
- Workspace switcher (single workspace assumption)
- Drag reorder (FEAT-009 ships SortableJS hook generically; sidebar is a reorder target but actual DnD wiring lives there)
- Page metadata editing (FEAT-013)
- Block rendering (FEAT-008+)
- Search functionality (FEAT-014)

* Existing Patterns
- =PetalComponents.Sidebar=, =Dropdown=, =Modal=, =Field=, =Button=
- LV streams for the tree set: =stream(:pages, list)= keyed by id; on PubSub update use =stream_insert= / =stream_delete=
- =Phoenix.Component.live_component= for =PageTree= so it owns local UI state (expanded set)
- Navigation via =push_patch=

* Tests
File: =test/concept_web/live/workspace_live_test.exs=

Scenarios:
1. Authenticated user lands on =/w/:slug=; sidebar shows their workspace name + emoji + 0 pages.
2. Click "+ New page" → page created, pushed_patch to =/w/:slug/p/:page_id=, sidebar shows new row.
3. Hover a page row, click "+" → creates child page, parent auto-expands, child visible.
4. Double-click title → inline input; type "Roadmap" + Enter → page row updates, DB record updated.
5. Petal Dropdown → "Archive" → row disappears; archived_at set in DB.
6. Two open browser tabs (simulated via two LV processes in test) — creating a page in tab A appears in tab B's sidebar within one PubSub round-trip.
7. Unauthenticated visit redirects to =/sign-in=.
8. Membership absent: visiting another user's =/w/:other_slug= → redirect to own workspace with flash error.

* Edge Cases
- Slow DB / many pages: cap initial sidebar load at 500 rows; show "Show more" if exceeded (FUP for virtualized list).
- Inline rename racing with PubSub update from another tab: last-write-wins; show subtle flash if conflict detected.
- Archiving currently-open page: redirect to =/w/:slug= empty state with flash.
- Browser back/forward: =push_patch= handles state correctly.
- Workspace slug with unicode: route uses URL-encoded slug; rejected if not ascii in current MVP (slug generator already enforces).
