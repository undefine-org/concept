* Intention & Purpose
Promote `CommandPaletteLive` from title-only fuzzy search to hybrid: existing keyword title search PLUS `Concept.Knowledge.search` (vector + graph fusion). Results merge, dedupe by page, and rank with title hits prioritized first within a small budget, then top semantic hits.

* Context & Narrative
Existing `lib/concept_web/live/command_palette_live.ex` calls `Concept.Pages.search_titles(query)` to populate results. We add a parallel call to `Concept.Knowledge.search`. Merging logic: title hits dedupe with semantic hits at the (page_id) level; the FIRST occurrence wins (so a title hit shadows a deeper block hit on the same page). Each result stores `:source` (`:title | :semantic`) so the UI can show a small indicator.

Latency budget: 350ms for whole palette refresh. Title search is local; semantic search ~150-300ms (Gemini embed query + vector search). We run both concurrently via `Task.async_stream` and accept partial results if semantic is slow.

* Scope
- IN: parallel `Task.async_stream` over [title_search, semantic_search]; result merger; UI: snippet + breadcrumb + source indicator; navigation to `/w/<slug>/p/<page>#block-<bid>` for semantic hits with block_id.
- OUT: typo-tolerant title search (Postgres trigram); per-result preview popover.

* Edge Cases
- `Concept.Knowledge.search` not yet ingested any pages → returns empty list, palette degrades to title-only.
- Workspace just created (no collection): `Collections.ensure_for_workspace` is called by ingest paths only; semantic search hitting a missing collection should return empty list gracefully (Search module catches `Ecto.NoResultsError` → `[]`).
- Very long query (>500 chars): truncate to 500 client-side AND server-side before passing to Gemini.
- Hit with stale `block_id` (block was archived after indexing, before reindex): clicking still navigates; PageEditorLive shows page, anchor not found, hook silently no-ops (FEAT-025 edge case). FUP: filter dead block hits via Ash query.
- Concurrent typing: cancel previous async tasks via `cancel_async/2`.
