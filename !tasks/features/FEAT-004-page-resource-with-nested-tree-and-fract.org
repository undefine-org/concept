* Initial Message

implement a pixel-perfect version of Notion. Use Ash as much as possible. This plan should cover the MVP version.

* Intention & Purpose
Model Notion-style nested pages as =Concept.Pages.Page= within =Concept.Pages= domain. Pages are tree-structured per workspace via =parent_page_id= self-FK, ordered by fractional-index =position= (LexoRank-like string), soft-deletable via =archived_at=. Provide actions: =:create_page=, =:rename=, =:set_icon=, =:set_cover_color=, =:reparent=, =:reorder=, =:archive=, =:restore=. Authorisation: actor must be a member of the workspace.

* Context & Narrative
Foundation for sidebar tree (FEAT-005) and block storage (FEAT-006). Fractional indexing avoids re-shuffling siblings on insert; we use string keys (LexoRank single-char alphabet) so two clients can insert "between A and B" by midpoint without coordination.

* Scope
** In-scope
- =Concept.Pages= Ash domain with =code_interface= exposing: =create_page!/1=, =rename_page!/2=, =set_icon!/2=, =set_cover_color!/2=, =reorder_page!/2=, =reparent_page!/3=, =archive_page!/1=, =restore_page!/1=, =list_tree!/1=, =get_page!/1=, =breadcrumbs!/1=
- =Concept.Pages.Page= Ash resource with extensions =AshPostgres.DataLayer=, =AshArchival.Resource=, =AshPaperTrail.Resource=
  - Multitenancy: =multitenancy do; strategy :attribute; attribute :workspace_id; global? false; end= — every read/write is workspace-scoped automatically
  - Attributes: =id (uuid), workspace_id (FK, tenant), parent_page_id (nullable self-FK), title (:string, default ""), icon_emoji (:string, default "📄"), cover_color (:string, constraint one_of), position (:string, fractional index)=
  - Soft-delete via =AshArchival= (=archived_at= managed automatically; =:destroy= becomes archive)
  - Version history via =AshPaperTrail= → table =page_versions=
- =Concept.Pages.FractionalIndex= module: =between/2=, =after/1=, =before/1=, =initial/0=
- Actions:
  - =:create_page= args =parent_page_id (optional), title=; computes position; =workspace_id= comes from tenant
  - =:rename= arg =title=
  - =:set_icon= arg =icon_emoji=
  - =:set_cover_color= arg constrained to =~w(default red orange yellow green blue purple pink gray)=
  - =:reparent= args =parent_page_id, position=
  - =:reorder= arg =position= (within current parent)
  - =:archive= via AshArchival (cascades to descendants in transactional change; descendants archived_at is set to same timestamp)
  - =:restore= clears =archived_at=
- Calculations: =calculate :depth, :integer, expr(...recursive...)=, =calculate :breadcrumbs, {:array, :map}, Concept.Pages.Calculations.Breadcrumbs=
- Aggregates: =aggregate :children_count, :count, :children=, =aggregate :block_count, :count, :blocks= (FK exists from FEAT-006; declare as relation upfront with empty target until FEAT-006 lands — feature-flag the aggregate behind a compile-time guard or stub it as 0 until FEAT-006)
- Read action =:list_tree= ordered by =position=; benefits from auto-archival filter
- Index =(workspace_id, parent_page_id, position)= for sidebar query

* Tests

Scenarios:
1. =FractionalIndex.between("a", "c") == "b"=; =between("a", "b") == "an"= (midpoint extending alphabet); =initial() == "n"= (mid of a..z).
2. Creating 3 pages at root produces strictly-increasing positions; sibling order matches insertion order.
3. =:reorder= moves page between siblings: target between B and C → new position is =FractionalIndex.between(B.pos, C.pos)=; siblings unchanged.
4. =:reparent= to deeper parent updates =parent_page_id=; descendants stay attached (subtree moves with it).
5. =:archive= on a parent cascades =archived_at= to all descendants in one transaction; =:list_tree= returns =[]=.
6. =:restore= clears archived_at on the page only (descendants stay archived unless explicitly restored).
7. Policy: a user not in the workspace gets =Ash.Error.Forbidden= on =:create_page=.
8. Cycle prevention: =:reparent= where new parent is a descendant raises =:cycle_detected= validation error.

- =lib/concept/pages/fractional_index.ex= with alphabet =?a..?z=, BASE=26.
- =between(nil, nil)= = =initial/0= = =<<(?a + 12)>>= (≈"m").
- =between(left, right)= midpoint by treating strings as base-26 fractions; recurse on next char if midpoint equals one bound.

* Edge Cases
- Position string overflow: deep midpoints can grow unbounded. Periodic compaction is FUP; document in module doc.
- Concurrent reorder of same page from two clients: last-writer-wins on position; acceptable for MVP.
- Archiving root-level page hides whole subtree from sidebar but preserves data; restoring root only restores root (descendants need explicit restore — keeps logic simple).
- Workspace deletion: out-of-scope (no cascade needed yet).
