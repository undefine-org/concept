* Intention & Purpose
Center of gravity for all AI answers. Use ~mix ash_ai.gen.chat --live~ to generate ~Knowledge.Conversation~ + ~Knowledge.Message~ resources (state machines, AshOban backing, PubSub streaming, LiveView scaffolding). Extend the generated Message resource with Concept-specific fields and link to Citation (FEAT-032). Streaming via ~AshAi.ToolLoop.stream/2~ broadcasts tokens on ~conversation:<id>~ topic — no custom answerer.

* Scope (IN)
- Run ~mix ash_ai.gen.chat --live~. Move generated files into ~lib/concept/knowledge/~ namespace.
- Generated resources (AshAI):
  - ~Concept.Knowledge.Conversation~: id, workspace_id (tenant), user_id, title (LLM-generated from first message), last_message_at, archived.
  - ~Concept.Knowledge.Message~: id, conversation_id, role (~:user | :assistant | :tool~), content, tool_calls jsonb, tool_results jsonb, model, state SM (~:pending → :streaming → :complete | :failed | :cancelled~).
- Extensions we add to ~Message~:
  - ~scope~ attribute (~:workspace | :page | :subtree | :block~)
  - ~scope_target_id~ attribute (uuid|nil)
  - ~profile~ attribute (atom — refs Spark profile in FEAT-034)
  - ~search_trace~ attribute (jsonb — top-k pre-rerank chunks with scores)
  - ~rewritten_prompt~ attribute (string)
  - ~latency_ms~, ~prompt_tokens~, ~completion_tokens~ attributes
  - ~has_many :citations~ (FEAT-032)
- Domain-level ~tools do ... end~ block exposing:
  - ~tool :search_workspace, Knowledge, :search~
  - ~tool :answer_question, Knowledge, :ask~
  - ~tool :create_page, Pages.Page, :create_page~
  - ~tool :link_blocks, Knowledge.Link, :create~
  - ~tool :summarize_page, Pages.Page, :summarize_subtree~
- Pre-LLM retrieval step happens inside the ~:search~ action; AshAI's loop then sees results as tool output.
- Multitenant by workspace_id; system actor bypass.
