* Intention & Purpose
Mount =AshAdmin= at =/admin= as a workspace-owner-only resource explorer. Useful for dev/staging inspection of Users, Workspaces, Memberships, Pages, Blocks, PaperTrail rows, Oban jobs.

* Context & Narrative
AshAdmin is a one-line install that auto-discovers Ash domains and exposes a CRUD UI honoring policies. The only custom work here is the auth guard.

* Scope
** In-scope
- =/admin= route through =AshAdmin.Router=, scoped behind a =:require_owner_or_admin= plug
- =:require_owner_or_admin= plug checks =current_user= is membership-owner of at least one workspace
- =Concept.Accounts= and =Concept.Pages= domains exposed in admin via =admin do; show? true; end= block on each domain (or per-resource toggle)
- Read-only by default for Block content (jsonb is huge); table view shows truncated content
- Test: unauthenticated → 302 to /sign-in; member non-owner → 403; owner → 200

** Out-of-scope
- Per-resource permission tuning beyond default (FUP)
- Custom admin actions / dashboards (FUP)

* Existing Patterns
- =AshAdmin.Router= macro
- Phoenix plug pipelines

* Tests
File: =test/concept_web/admin_test.exs=. Three scenarios.
