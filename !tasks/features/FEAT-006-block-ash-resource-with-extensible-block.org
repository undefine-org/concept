* Intention & Purpose
Establish the single =Concept.Pages.Block= Ash resource (multitenant, archival, paper-trailed, state-machine-locked) and the extensible =Concept.Pages.BlockType= behaviour + explicit-config registry. After this item: any LV/test can create/read/update/delete blocks of any type via Ash code interface; type validation and Lexical content shape are owned by per-type modules; lock state transitions are guarded by a state machine; adding the 15th type is "drop a module + append to config".

* Context & Narrative
This is the keystone resource. Settled design (per plan-mode discussion):
- Single Ash resource → one table, uniform locks/policies/notifications/queries.
- =BlockType= behaviour → per-type concerns (Lexical schema, validation, slash-menu metadata, static render) live in plug-in modules.
- Explicit config registry: =config :concept, :block_types, [Paragraph, Heading1, ...]=.
- Custom Ash type =Concept.Pages.BlockTypeAttr= validates =:type= attribute against the registry at the data layer.

Block types defined in this FEAT (just paragraph + headings to prove the pipeline; remaining 12 types in FEAT-010 and FEAT-012):
- =Concept.Pages.BlockTypes.Paragraph=
- =Concept.Pages.BlockTypes.Heading1=, =Heading2=, =Heading3=

Scenarios:
1. Registry: =BlockTypes.all/0= equals config list; =lookup(:paragraph) == Paragraph=; =lookup(:unknown)= raises =:unknown_block_type=.
2. Custom Ash type: =:create_block= with =type: :paragraph= succeeds; =type: :nope= raises =Ash.Error.Invalid=.
3. =:create_block= without =position= computes after-last-sibling; with explicit position uses it.
4. =:update_content= without lock raises =Ash.Error.Forbidden= (state machine guard).
5. =:acquire_lock= sets state =:locked=, holder, acquired_at, expires_at = now + 30s; second user's =:acquire_lock= when state =:locked= raises =:lock_held_by_other=.
6. =:release_lock= clears holder/timestamps and transitions to =:unlocked=.
7. =:refresh_lock= keeps state =:locked=, extends =lock_expires_at= by 30s.
8. =BlockTypes.Paragraph.default_content/0= returns valid Lexical paragraph EditorState; =Concept.Lexical.plain_text(content) == ""=.
9. =:update_props= with invalid props for paragraph (paragraph has no props — must be empty map) returns validation error.
10. AshPaperTrail: editing content writes a row to =block_versions= with =changes: %{content: _}=.
11. Multitenancy: blocks of page in workspace A are invisible to workspace B's actor.
12. Archival: =:destroy= soft-deletes; =:list_for_page= excludes archived.
13. Cycle prevention on =:reparent= (when parent_block_id chain forms cycle).
14. PubSub: creating a block publishes ={:block_created, block}= on =workspace:#{ws}:page:#{page}:blocks=.

* Edge Cases
- Two clients race =:acquire_lock=: one transition succeeds (state machine + Postgres row lock via =Ash.Changeset.before_action= takes =FOR UPDATE=); other gets =:lock_held_by_other=.
- Lock holder disconnects without releasing: TTL expiry handled by FEAT-007 AshOban trigger; until trigger fires, block is held — acceptable since TTL is 30s.
- Block type removed from config but exists in DB: =BlockTypes.lookup= raises; reads should degrade gracefully → render as paragraph fallback. Add =safe_lookup/1= that returns =Paragraph= on miss for static rendering only.
- Lexical EditorState is large (>1MB): jsonb fine up to ~Postgres TOAST limits; warn if =byte_size(Jason.encode!(content)) > 200_000= via validation.
- Reparenting block to its own descendant: cycle check.
- Cross-page reparent: blocked — =parent_block_id= must belong to same =page_id=.
- Hard delete via Ash.destroy with =:bypass_archive=: locked behind admin-only action (FUP).
