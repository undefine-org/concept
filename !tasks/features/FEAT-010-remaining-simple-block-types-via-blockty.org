implement a pixel-perfect version of Notion. Use Ash as much as possible. This plan should cover the MVP version.

* Intention & Purpose
Implement 9 additional non-composite block types as plug-in =Concept.Pages.BlockType= modules: bulleted_list_item, numbered_list_item, to_do, quote, divider, code, callout, toggle, image, bookmark, equation. Adding them touches:
- one Elixir module per type under =Concept.Pages.BlockTypes.*=
- one config line appending the module to =:block_types=
- (optional) Lit sub-components for non-text blocks (divider, image, bookmark, equation, to_do checkbox)

The Block Ash resource, lock manager, slash menu, drag handles, and PubSub from prior FEATs are unchanged.

* Context & Narrative
Tests the extensibility claim: zero edits to =Concept.Pages.Block= or =BlockEditor= hook. Each new type registers default content + props + render + slash menu metadata; container blocks (toggle) override =container?/0= and the renderer descends.

* Edge Cases
- ToDo checkbox while another user holds lock: user sees clickable but server rejects =:update_props= with lock error.
- Code block language list missing entry: server validation rejects unknown language.
- Image URL invalid (not http/https): props validation rejects.
- Bookmark URL with no protocol: prepend =https://=.
- Toggle collapsed with children that hold locks elsewhere: locks unaffected; rendering hidden but lifecycle preserved.
- Equation TeX with parse error: KaTeX renders error message inline; props persist as typed.
- Divider in middle of paragraph (cursor on divider): keyboard nav passes through (treated as block-level).
