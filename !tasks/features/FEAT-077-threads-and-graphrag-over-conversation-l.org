* Goal
Threads as child conversations (self-referential tree), and GraphRAG assembled
as a union over the host subgraph + the conversation lineage.

* Scope
1. ~Conversation~: add ~parent_conversation_id~ (self-ref) + ~seed_message_id~
   (the message a thread spawned from). A thread inherits the parent's host.
2. ~discuss~ gains the third resolution arm: ~reply_to_message_id~ → spawn a
   child conversation (same host, parent + seed set). This is the arm that
   provably writes two resources → promote ~discuss~ to a Reactor here
   (AGENTS.md rule 3).
3. GraphRAG union in retrieve_context: ~host.subgraph_scope(record)~ ∪
   ~conversation:<this>~ ∪ ~conversation:<each ancestor>~ (walk
   parent_conversation_id) ∪ extra-host scopes (future internal agents). Requires
   ~Search~ to accept a UNION of source scopes (extend search.ex — the one
   genuinely new retrieval capability, foreshadowed in design §32).
4. Title: fast-model generation (reuse today's ~generate_name~ /
   ~needs_title~ trigger) OR creator-set; threads inherit/derive a subject.

* Edge cases
- Lineage walk must be bounded (max depth) to avoid pathological ancestor chains.
- A thread's host = parent's host; do not re-resolve from scratch.
- Union retrieval must dedupe chunks shared across scopes and respect limit.
