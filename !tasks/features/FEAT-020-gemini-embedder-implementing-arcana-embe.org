* Intention & Purpose
Provide an Arcana-compatible embedder that hits Google's `text-embedding-004` API via Req. This is the substrate for every chunk's vector and every search query's vector. Performance and correctness here dominate retrieval quality.

* Context & Narrative
Arcana ships `Arcana.Embedder.Local` (Bumblebee) and `Arcana.Embedder.OpenAI` and exposes a behaviour for custom impls (`Arcana.Embedder` → callbacks `embed/2`, `embed_batch/2`, `dimensions/1`). Concept does not want a Bumblebee model in-process and does not want OpenAI. We implement a thin Req client to Gemini's batch embed endpoint at `https://generativelanguage.googleapis.com/v1beta/models/text-embedding-004:batchEmbedContents`.

`text-embedding-004` returns 768-dimensional vectors. The endpoint accepts up to 100 texts per request. Each request body element supports a `taskType` hint (`RETRIEVAL_DOCUMENT` for ingest, `RETRIEVAL_QUERY` for queries, `SEMANTIC_SIMILARITY` etc.). Arcana passes `intent: :document | :query` in opts — we map.

* Scope
- IN: `Concept.Knowledge.GeminiEmbedder` module implementing the behaviour; HTTP client logic w/ retry + exponential backoff via Req's built-in retry; intent → taskType mapping; batch chunking (splits inputs into ≤100-element chunks); dimensions/1 returns 768; telemetry spans.
- OUT: caching, multi-model selection, Bumblebee fallback. Hard-fail without `GOOGLE_API_KEY` per decision.

** ITEM Define Concept.Knowledge.GeminiEmbedder module
:PROPERTIES:
:CUSTOM_ID: FEAT-020-gemini-embedder-implementing-arcana-embe::module
:DEPENDS: FEAT-020-gemini-embedder-implementing-arcana-embe::tests

- File: `lib/concept/knowledge/gemini_embedder.ex`.
- `@behaviour Arcana.Embedder`.
- `@base_url "https://generativelanguage.googleapis.com/v1beta"`.
- `@model "models/text-embedding-004"`.
- `@max_batch 100`.

  ```elixir
  defmodule Concept.Knowledge.GeminiEmbedder.Error do
    defexception [:status, :body]
    @impl true
    def message(%{status: s, body: b}), do: "Gemini embed failed (status=#{s}): #{inspect(b)}"
  end

  @impl Arcana.Embedder
  def dimensions(_opts), do: 768

  @impl Arcana.Embedder
  def embed(text, opts) when is_binary(text) do
    with {:ok, [vec]} <- embed_batch([text], opts), do: {:ok, vec}
  end

  @impl Arcana.Embedder
  def embed_batch(texts, opts) do
    intent = Keyword.get(opts, :intent, :document)

    texts
    |> Enum.chunk_every(@max_batch)
    |> Enum.reduce_while({:ok, []}, fn chunk, {:ok, acc} ->
      case do_request(chunk, intent) do
        {:ok, vecs} -> {:cont, {:ok, acc ++ vecs}}
        {:error, _} = err -> {:halt, err}
      end
    end)
  end

  defp do_request(texts, intent) do
    :telemetry.span([:concept, :knowledge, :embedder, :gemini], %{count: length(texts)}, fn ->
      url = "#{@base_url}/#{@model}:batchEmbedContents?key=#{Concept.Knowledge.Config.api_key()}"
      body = %{requests: Enum.map(texts, &request_for(&1, intent))}

      case Req.post(url,
             json: body,
             retry: :transient,
             max_retries: 3,
             retry_delay: fn n -> trunc(:math.pow(2, n) * 200) end
           ) do
        {:ok, %{status: 200, body: %{"embeddings" => embeds}}} ->
          vecs = Enum.map(embeds, fn %{"values" => v} -> v end)
          {{:ok, vecs}, %{result: :ok}}

        {:ok, %{status: status, body: body}} ->
          {{:error, %__MODULE__.Error{status: status, body: body}}, %{result: :error}}

        {:error, exception} ->
          {{:error, exception}, %{result: :error}}
      end
    end)
  end

  defp request_for(text, intent) do
    %{
      model: @model,
      content: %{parts: [%{text: text}]}
    }
    |> maybe_put_task_type(intent)
  end

  defp maybe_put_task_type(req, :document), do: Map.put(req, :taskType, "RETRIEVAL_DOCUMENT")
  defp maybe_put_task_type(req, :query), do: Map.put(req, :taskType, "RETRIEVAL_QUERY")
  defp maybe_put_task_type(req, :similarity), do: Map.put(req, :taskType, "SEMANTIC_SIMILARITY")
  defp maybe_put_task_type(req, _), do: req
  ```

* Edge Cases
- Empty string input: Gemini rejects empties with 400. We forward the error untouched; chunker (FEAT-021) is responsible for never emitting empty chunks.
- Network timeout: Req's default 15s; on persistent failure retry 3× then return `{:error, exception}`. Caller (AshOban worker) decides retry policy.
- 429 rate limit: Req's `retry: :transient` covers 429 (verify Req version handles it; if not, set `retry: :safe_transient` or custom).
- 4096-token limit per text: Gemini's max is 2048 tokens. Larger inputs return 400. BlockChunker (FEAT-021) targets ≤512-token chunks; we do not pre-truncate here.
- Vector dimension mismatch: if Google rolls a new model dim, our migration's `vector(768)` column will reject inserts. Pin via `@model` constant + `dimensions/1`.
