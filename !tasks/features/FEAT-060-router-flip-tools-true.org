* Intention & Purpose

Drop the explicit ~tools: [...]~ whitelist in favor of ~tools: true~.
After this FEAT, every described action across every domain is reachable
via MCP. Workspace context is enforced by FEAT-058's plug.

* Scope (IN)

** ~lib/concept_web/router.ex~

#+begin_src elixir
scope "/mcp" do
  pipe_through :mcp

  forward "/", AshAi.Mcp.Router,
    tools: true,
    otp_app: :concept,
    protocol_version_statement: "2025-03-26",
    tenant: {ConceptWeb.Plugs.MCPWorkspaceContext, :fetch_tenant, []}
end
#+end_src

* Implementation notes

- ~tools: true~ is interpreted by ~AshAi.exposed_tools/1~ as "return every
  tool declared across all domains in :ash_domains". This combined with
  ~Concept.AutoTools~ (FEAT-052) means: every described action, exposed.
- Confirm the ~:tenant~ option shape AshAi.Mcp.Router accepts in the
  installed version. If MFA isn't supported there, fall back to setting
  the tenant via Plug ~assign~ and have AshAi pick it up from
  ~conn.assigns[:tenant]~ — check the installed code path before locking
  in the syntax.

* Edge cases

- If a domain has an action that should be unexposed but accidentally got
  a description in W1 → it appears in ~tools/list~. Mitigation: FEAT-061's
  parity test fails CI if any "leaked" description is detected; add the
  action's name to a project-level deny list (~config :concept, :mcp_deny_tools, [...]~)
  or fix the description in source.

* Acceptance

- ~tools/list~ returns all expected tools.
- ~tools/call~ on a read tool (e.g. ~page_list_tree~) returns the same
  result as the equivalent code interface call (~Concept.Pages.list_tree~).
