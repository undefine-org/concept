* Goal
Dissolve the content-layer membrane (a Message contains Blocks) AND generalize
the page-keyed ingest pipeline to a ~{source_type, id}~ ref. The crystallization
pressure-test (docs/messaging_design.md Part VIII) proved this generalization is
LOAD-BEARING for three features at once: message ingestion, conversation-scoped
RAG, and crystallize provenance. This is the spine of "conversation is knowledge".

* Scope
1. ~Block~ gains nullable ~message_id~ + ~belongs_to :message~. Postgres CHECK
   ~num_nonnulls(page_id, message_id) = 1~ (a block lives under a page XOR a
   message). ~references~ + index on ~[:workspace_id, :message_id, :position]~.
2. ~Message.blocks~ has_many (destination ~message_id~). Keep ~Message.text~ as
   a fast-path shim (plain-text mirror) for now; ~blocks~ is the rich body.
3. ~AssignAfterLastSibling~: generalize sibling query to key off the container
   (page_id OR message_id), not page_id alone.
4. source_id generalization (page → container ref):
   - ~KnowledgeReindex~ block clause dispatches on container: page_id →
     ~enqueue(:page, id)~; message_id → ~enqueue(:conversation, conv_id)~
     (conversation is the natural ingest grain). Guard the nil-page_id crash.
   - ~IngestPage~ → ~IngestSource~ (or sibling worker): for a conversation
     source, list the conversation's message-blocks instead of ~list_for_page~;
     ~source_id: "conversation:<id>"~.
   - ~Search.scope_opts~ gains ~conversation:<id>~ (≡ the GraphRAG union term).
   - debounce ~unique~ key: ~[:page_id, :op]~ → ~[:source_type, :source_id, :op]~
     so non-page sources dedupe per-source (nil-page collision fix).
   - ~Citation~ block/page FKs already on_delete:delete; grounding citations are
     block-addressed and unaffected.

* Edge cases

- A message-block reparented to a page (Wave 5 crystallize) crosses container

  the new page source (extend the existing cross-page dual-ingest path).

  BlockRender dispatcher; messages render the same block kinds as pages).

** Process learnings (important for remaining waves)
- The shared tree has HEAVY concurrent-agent activity. Build-lock races
  (`Waiting for lock … held by process NNNN`) and _build/test clobbering produce
  TRANSIENT, misleading failures: false EXIT=1 from --warnings-as-errors and a
  phantom "Block does not have field message_id" Spark error mid-run. The fix:
  `MIX_ENV=test mix compile --force` then re-run; verify in an ISOLATED run.
  Source correctness was provable via `git diff HEAD` (empty) + a quiet
  force-compile (EXIT=0).
- Parallel edit calls to the SAME file race and corrupt it (duplicate
  belongs_to, reverted attrs). Rule for remaining waves: ONE atomic multi-op
  edit per file; verify on disk before the next step; never batch edits to a
  hot file with concurrent writers active.
- Recover corrupted files via `git show <commit>:path > path` (read+write,
  never checkout/reset — per the hard no-destructive-git rule).
