* Intention & Purpose
Embed AI answers as a first-class block type. The block's content stores ~{query_id, model, ran_at}~ — a pointer to the durable Knowledge.Query. Generate button creates a Query (scope ~:subtree~ by default); tokens stream into the block live; on finalize, the block updates its pointer atomically. Refresh re-runs.

* Scope (IN)
- ~Concept.Pages.BlockTypes.AiAnswer~ already exists; this feature gives it a body.
- Block content schema: ~%{"message_id" => uuid, "model" => string, "ran_at" => iso8601}~ — points at an AshAI Message row (the durable answer + citations live there).
- Block props: ~%{"prompt" => string, "scope" => "subtree"|"page"|"workspace", "profile" => "fast"|"default"|"thorough"}~.
- ~Concept.Pages.Block~ gains action ~:evaluate_ai~ — implemented via ~AshAi.Actions.Prompt~ for the synchronous one-shot path; for streaming, wraps ~AshAi.ToolLoop.stream/2~ in an AshOban worker that:
  1. Creates an internal Conversation (or reuses a "block-conversations" one per block).
  2. Adds a user Message with the prompt + scope context.
  3. Streams via ToolLoop with profile-selected tools.
  4. On completion, writes block content pointer ~{message_id, model, ran_at}~ via atomic update.
- Lit web component ~ora-ai-block~ renders 4 modes: ~empty~ (prompt + Generate button), ~streaming~ (subscribed to ~conversation:<id>~ topic via PubSub bridge hook, shows live tokens), ~answered~ (markdown answer + collapsed CitationCard strip + Refresh), ~failed~ (error + Retry).
- ~ConceptWeb.BlockRender~ branch for ~:ai_answer~ instantiates ~ora-ai-block~ with the right phx-hook; loads referenced Message via Ash if content has ~message_id~.
