* Intention & Purpose
Promote ad-hoc per-page reindexing into a tracked Ash resource. Every ingest attempt is a row with state, timing, chunk counts, and errors. AshOban triggers run queued rows. PubSub broadcasts transitions for the IndexingPill UI.

* Scope (IN)
- Resource ~Concept.Knowledge.IngestionJob~ with attributes: ~workspace_id~ (tenant), ~page_id~, ~op~ (~:upsert | :delete~), ~state~ (~:queued | :running | :succeeded | :failed~), ~scheduled_at~, ~started_at~, ~finished_at~, ~chunk_count~, ~embed_tokens~, ~error_kind~, ~error_message~, ~attempt~.
- ~ash_state_machine~ with the four transitions.
- ~ash_archival~ for soft-delete of completed jobs older than 30 days.
- ~pub_sub~ publishing ~workspace:<id>:ingest~ on every transition.
- AshOban trigger ~:run~ that picks up ~:queued~ rows, transitions to ~:running~, calls FEAT-035 Reactor, transitions to ~:succeeded~ or ~:failed~ with measurements.
- Multitenant by ~workspace_id~ (global? false).
- Policies: read for workspace members; write only for system actor.
- ~AshAdmin~ visibility.

* Tests (8 scenarios)
1. Creation requires workspace_id + page_id.
2. State machine rejects invalid transitions (~:queued → :succeeded~ without running).
3. PubSub broadcasts on every transition (~assert_receive~).
4. AshOban trigger ~:run~ picks up queued, processes, transitions.
5. Policy: non-member ~:read~ → forbidden.
6. Policy: ~:create~ by user (not system) → forbidden.
7. Archival: completed jobs > 30d soft-deleted.
8. Concurrent updates use ~require_atomic? false~ + optimistic concurrency.
