* Initial Message

how would you integrate Arcana into this, maximally leveraging Ash to make the system as simple as possible?

* Intention & Purpose
Owner-only operations surface for Arcana: mount the Arcana Dashboard at `/admin/arcana` and surface key Arcana telemetry events in the existing `/dev/dashboard` Phoenix LiveDashboard. Make ingestion and search behavior observable.

* Context & Narrative
Arcana ships a LiveDashboard-style UI (collections list, document inspector, evaluation results). It exposes a router macro for mounting (e.g. `arcana_dashboard "/arcana"`). Concept already has an owner-gated `/admin` scope (FEAT-016 mounted AshAdmin there with `:require_owner` plug).

For metrics: Arcana emits `[:arcana, :search]`, `[:arcana, :embedder, :embed]`, `[:arcana, :pipeline, :*]`, `[:arcana, :ingest, :*]`, `[:arcana, :graph, :*]` events. Concept's `ConceptWeb.Telemetry` module declares `metrics/0` consumed by both LiveDashboard and possibly Prometheus. We add summary/counter metrics for the most actionable events.

* Scope
- IN: router mount for Arcana Dashboard at `/admin/arcana` (owner-only); Telemetry.metrics extended with Arcana events; documentation comment in `ConceptWeb.Telemetry`.
- OUT: Prometheus exporter; Grafana dashboards.

* Existing Patterns (MUST follow)
- `lib/concept_web/router.ex`: `import AshAdmin.Router; scope "/admin" do pipe_through [:browser, :require_owner]; ash_admin "/" end`. Mirror — add a sibling line.
- `lib/concept_web/telemetry.ex` — extend `metrics/0`.
- LiveDashboard route is dev-only (`Application.compile_env(:concept, :dev_routes)`). Do NOT change that gating.

* Tests (file paths + scenarios)
Test files: `test/concept_web/dashboard_test.exs`, `test/concept_web/telemetry_test.exs`.
Scenarios:
1. GET `/admin/arcana` while NOT signed in → 302 to login.
2. GET `/admin/arcana` while signed in but NOT owner → 403.
3. GET `/admin/arcana` while signed in as owner → 200 with Arcana page rendered.
4. `ConceptWeb.Telemetry.metrics/0` returns at least 6 metric definitions whose names start with `arcana.`.
5. After triggering a search, the metric `arcana.search.duration` has a recorded value (read via `Telemetry.Metrics` helper).
