* Initial Message

Let's think. What is a nice, simple system that lets us do project management within Concept? As opposed to simple knowledge base stuff. This would mean:
- it's easy to know which tasks need to be done. which are known as tasks and which ones are aspirational/research/whatever.
- task assignment

look up online to read expert designers' takes on the shortcomings of the Notion model. This is a DESIGN session where we think deeply, from first principles, about what can help us achieve these principles:
1. Simple (it's easy for agents and humans to understand the system)
2. Transparent & not prone to redundancy (it's easy to find where each task is, and it's easy to know where to put each task)
4. Ease of management

For this first iteration, think about what we'd do if we had to design Notion's system from scratch, with the principle that Concept is for human and agent TEAMS working together. Ask me plenty of questions to operationalise this principle

New block type per docs/blocks/ADDING_A_BLOCK.md (one registry line in
config :concept, :block_types; zero dispatcher edits). props %{record_id}.
Renders the referenced Record's LIVE title, state (category color), assignee.
Editing status/assignee from inside a doc calls the same Record actions. This
is the projection that lets any page mention a record without copying it —
the structural redundancy-killer.
