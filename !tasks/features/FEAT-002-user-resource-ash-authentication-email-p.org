* Initial Message

implement a pixel-perfect version of Notion. Use Ash as much as possible. This plan should cover the MVP version.

* Intention & Purpose
Stand up =Concept.Accounts.User= as an Ash resource with =AshAuthentication= email+password strategy and integrate =AshAuthenticationPhoenix= LiveView routes for sign-in/sign-up/sign-out + a session-aware =on_mount= guard. Output: a visitor can register, log in, log out; protected routes redirect to =/sign-in=; =current_user= is available in every LiveView.

* Context & Narrative
Builds on FEAT-001 (Phoenix+Ash). This is the bedrock for FEAT-003 (workspaces auto-create on signup). Petal supplies the form chrome.

* Scope
** In-scope
- =Concept.Accounts= Ash domain
- =Concept.Accounts.User= resource: =id (uuid), email (citext), hashed_password, display_name, avatar_emoji, inserted_at, updated_at=
- =AshAuthentication= strategy =:password= (email + password, no magic link, no oauth)
- =Concept.Accounts.Token= resource (Ash-generated for revocation/sessions)
- =AshAuthenticationPhoenix= router macros (=sign_in_route=, =sign_out_route=, =reset_route=, =register_route=) under =/=. Custom Petal-styled overrides for sign_in.html and register.html (LV components).
- =ConceptWeb.LiveUserAuth= module with =on_mount :live_user_required= and =:live_user_optional=
- Migration via =mix ash.codegen --name install_users=
- LiveView tests for register/login/logout flow

** Out-of-scope
- OAuth providers, magic link, password reset email delivery (=Swoosh= is configured but emails go to local Mailbox preview only)
- Workspace creation logic (FEAT-003)
- Avatar upload (stub: emoji only)

* Existing Patterns
- AshAuthentication =Strategy.Password= docs: https://hexdocs.pm/ash_authentication/AshAuthentication.Strategy.Password.html
- Phoenix=AshAuthenticationPhoenix.Router=: =sign_in_route on_mount: [{ConceptWeb.LiveUserAuth, :live_user_optional}]=
- Petal: =<.field />= component for inputs, =<.button />= for submit; the AshAuthenticationPhoenix overrides expect =assigns.form= shaped like AshPhoenix.Form
- Use =AshPhoenix.Form= for any custom forms (later FEATs)

* Tests
Files: =test/concept_web/live/auth_live_test.exs=, =test/concept/accounts/user_test.exs=

Scenarios:
1. =register/0= action with valid email + password creates User, returns token; duplicate email errors with =:email_taken=.
2. =sign_in_with_password/0= rejects bad password with =:invalid_credentials=.
3. LV: =GET /sign-up= renders Petal form; submit with valid params redirects to =/= and =current_user= assigned.
4. LV: =GET /sign-in= happy path → redirect to =/=; bad creds → flash "Invalid email or password".
5. LV: =/sign-out= clears session; subsequent =GET /protected= → redirect to =/sign-in=.
6. =on_mount :live_user_required= halts unauthenticated mounts with =push_navigate to: "/sign-in"=.

* Edge Cases
- Email case-insensitive: =:citext= attribute type; lowercase normalisation in change.
- Password min length 8, validated via Ash validation.
- Token rotation: AshAuthentication revokes on sign-out.
- Concurrent logins from multiple tabs: tokens are independent, fine.
- Reset-password route exists but emails are dev-only =Swoosh.Adapters.Local= (FUP for prod email).
