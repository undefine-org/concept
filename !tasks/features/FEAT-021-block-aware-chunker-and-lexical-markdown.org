* Intention & Purpose
Chunking dictates citation precision and retrieval quality. We supply a custom Arcana chunker that mirrors Concept's block tree: each top-level block becomes one chunk (with metadata pointing to its block_id), tiny sibling blocks are coalesced up to a token threshold, and container blocks (`columns`, `toggle`, `table`) are flattened depth-first while preserving their semantic role. Output also feeds the deep-link citations.

* Context & Narrative
`Concept.Lexical.plain_text/1` already exists but emits no structure or markdown — useless for citations because we need block boundaries. We extend with `Concept.Lexical.to_markdown/1` (block-aware, mark-aware). Then `Concept.Knowledge.BlockChunker` walks the loaded page tree (page → top-level blocks → containers → children). Output: list of chunks, each with `:text`, `:chunk_index`, `:token_count`, `:metadata` (block_id, page_id, workspace_id, block_type, breadcrumbs).

Arcana's chunker behaviour: `@callback chunk(text :: String.t(), opts :: keyword) :: [chunk_map]`. We accept `opts` containing `[blocks: list, page: page_struct, workspace_id: id]` and IGNORE the `text` arg (we already have structured input from the ingest action). Arcana docs note this is acceptable for fully-custom chunkers — the chunker has full control.

* Scope
- IN: `Concept.Lexical.to_markdown/1` (with marks: bold, italic, underline, strike, code, links); `Concept.Knowledge.BlockChunker` rendering each block to markdown + assembling chunks; configurable thresholds (defaults: target ≤512 tokens, merge siblings <128 tokens); breadcrumb computation (page title chain).
- OUT: chunking PDFs/files (Arcana default chunker handles those if we ever need them).

* Existing Patterns (MUST follow)
- `Concept.Lexical` module already implements `plain_text/1` and `to_html/1`. Add `to_markdown/1` alongside, using same defp render_node/walk_text pattern.
- `Concept.Pages.BlockTypes` registry exposes `lookup/1` returning a module per block type. Use it to dispatch markdown rendering by type.
- `Concept.Pages.Block` has `children` aggregate via `parent_block_id`. To build a tree from a flat list, group by `parent_block_id` then recurse from `nil`-parents.
- `Concept.Pages.Page` has `parent_page_id`. Breadcrumbs = walk parents to root, collect titles.
- Token estimation: rough `byte_size(text) / 4` is fine for v1; precise tokenization deferred (FUP).

8. Re-chunking the same input is deterministic (stable order, identical metadata).

:PROPERTIES:
:CUSTOM_ID: FEAT-021-block-aware-chunker-and-lexical-markdown::chunker-impl

- File: `lib/concept/knowledge/block_chunker.ex`.
- `@behaviour Arcana.Chunker`.
- Module attributes: `@target_tokens 512`, `@merge_below 128`.
- Public API: `chunk(_text_unused, opts)` reads `opts[:blocks]`, `opts[:page]`, `opts[:workspace_id]`, `opts[:breadcrumbs]`; returns list of chunk maps.

  ```elixir
  defmodule Concept.Knowledge.BlockChunker do
    @behaviour Arcana.Chunker
    @target 512
    @merge_below 128

    @impl true
    def chunk(_ignored, opts) do
      blocks = Keyword.fetch!(opts, :blocks)
      page = Keyword.fetch!(opts, :page)
      workspace_id = Keyword.fetch!(opts, :workspace_id)
      breadcrumbs = Keyword.get(opts, :breadcrumbs, page.title)

      tree = build_tree(blocks)

      tree
      |> Enum.map(&render_block(&1, %{depth: 0}))
      |> Enum.reject(&(String.trim(&1.text) == ""))
      |> coalesce_small()
      |> Enum.with_index(0)
      |> Enum.map(fn {%{text: t, block_ids: ids, primary_id: pid, type: type}, idx} ->
        %{
          text: t,
          chunk_index: idx,
          token_count: max(1, div(byte_size(t), 4)),
          metadata: %{
            "page_id" => page.id,
            "workspace_id" => workspace_id,
            "block_id" => pid,
            "block_ids" => ids,
            "block_type" => to_string(type),
            "breadcrumbs" => breadcrumbs
          }
        }
      end)
    end

    # build_tree/render_block/coalesce_small implementations: see test fixtures.
  end
  ```
- `render_block` renders the block via `Concept.Lexical.to_markdown/1`, prepends type-specific prefix (`# `, `## `, `> `, `- `, etc.), and recursively renders children indented by 2 spaces per depth level for containers.
- `coalesce_small`: walk siblings; if running buffer + next < @target AND next < @merge_below, append; else flush.

* Edge Cases
- Block with empty Lexical content (`%{}`): `to_markdown/1` returns `""`; chunker drops it (scenario 6).
- Container with no children (`columns` count=0): renders an empty placeholder chunk → dropped.
- Very long single block (paragraph > target tokens): emit as ONE chunk anyway (do not split mid-block; downstream Arcana default chunker would split text but we override). Note in moduledoc.
- Unknown block_type: fall back to `Concept.Lexical.plain_text/1` only.
- Cyclic block tree (parent_block_id loop) — already prevented by `Concept.Pages.Changes.PreventCycles` for pages; for blocks add a loop guard (`MapSet` of seen ids) raising on detection.
