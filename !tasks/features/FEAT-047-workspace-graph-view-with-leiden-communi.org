* Intention & Purpose
Visualize Concept's self-built knowledge graph. Force-directed view of pages + (optionally) blocks, colored by Leiden community (from Arcana.Maintenance.detect_communities), edge thickness from authored Link + structural CONTAINS/PARENT_OF counts.

* Scope (IN)
- New route ~live "/w/:workspace_slug/graph", WorkspaceGraphLive~.
- Server-side: build a serialized graph payload via ~Knowledge.graph_for_workspace/2~ (reads ~Arcana.Graph.Entity~ + ~Relationship~ + community summaries).
- Client-side: Lit element ~ora-workspace-graph~ uses d3-force imported into app.js (no external script tags). Renders SVG.
- Node click → side panel with community summary + member pages list.
- Filters: by block_type, by community, by "ego graph from page X" (uses ~Arcana.Graph.fusion_search~ neighborhood query).
- V1 limits: pages-only view by default; "show blocks" toggle off by default. Block view for FUP if perf becomes an issue.
- Scheduled weekly community rebuild: AshOban global cron in supervisor → ~Concept.Knowledge.Workers.RebuildCommunities.perform_all_workspaces/0~.
