* Initial Message

implement a pixel-perfect version of Notion. Use Ash as much as possible. This plan should cover the MVP version.

* Intention & Purpose
Enable inline text formatting on every text-bearing block (paragraph, headings, list items, quote, callout, toggle, to_do): bold, italic, underline, strikethrough, inline code, and link (URL editor). Floating Petal-styled toolbar appears above text selection. Keyboard shortcuts: Cmd-B/I/U, Cmd-Shift-X, Cmd-E, Cmd-K (link).

* Context & Narrative
Lexical handles marks via =TextNode.format= bitmask natively. Link is its own node from =@lexical/link=. We add a Lit floating toolbar Lit element rendered when a Lexical selection is non-collapsed.

* Scope
** In-scope
- Lexical formats enabled in =assets/js/lexical/registry.js=: =BOLD=, =ITALIC=, =UNDERLINE=, =STRIKETHROUGH=, =CODE=. =LinkNode= from =@lexical/link= registered.
- =assets/js/components/ora-format-toolbar.js= Lit element: floats above selection (computed via getBoundingClientRect of selection); buttons toggle each format; Link button opens =ora-link-editor= popover.
- =assets/js/components/ora-link-editor.js= Lit popover: Petal-styled URL field + Apply/Remove buttons; on Apply runs =$toggleLink(url)= from =@lexical/link=.
- Keyboard shortcuts registered via Lexical command listeners in =BlockEditor= hook.
- Style: =.text-bold{font-weight:600}=, =.text-italic{font-style:italic}=, etc. via Lit element shadow CSS or global Tailwind utilities.
- Server-side static render: =BlockType.render_static/1= must walk Lexical TextNode format bits and emit corresponding HTML (=<strong>=, =<em>=, =<u>=, =<s>=, =<code>=, =<a href>=). Implemented in =Concept.Lexical.to_html/1= helper from FEAT-006, extended here.
- Tests: format toggling persists in saved =content=; static render emits correct tags.

** Out-of-scope

- Comments / suggestions on marks — FUP
- Mention nodes (@user) — FUP

* Edge Cases
- Multiple formats overlapping (bold+italic+code): both bits set; HTML nests =<code><strong><em>...=.
- Link on selection that already contains link: replace.
- Link URL invalid (no scheme, no domain): client warns, server validates to_html escapes URL.
- XSS via link href starting with =javascript:=: sanitize in to_html — only allow http/https/mailto.
- Cmd-K conflicting with browser shortcut: prevent default only when block is focused.
- Selection across multiple blocks: out of scope (single-block scope only); toolbar hides if selection spans blocks.
