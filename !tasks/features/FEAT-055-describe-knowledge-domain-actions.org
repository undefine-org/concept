* Context (read in full)

** The principle being enforced

A public Ash action with a ~description~ is automatically projected as an
MCP tool by ~Concept.AutoTools~ (FEAT-052). Descriptions are LLM-visible
and human-visible. Internal/system actions are left undescribed.

** Style guide

- One sentence, < 110 chars where possible.
- Imperative for mutations ("Create a relationship..."); declarative for reads ("List citations...").
- Domain language. No implementation jargon.
- Per-arg descriptions describe meaning + constraints.

#+begin_src elixir
argument :workspace_id, :uuid,
  description: "Workspace scope. Read isolation is enforced by tenancy."
#+end_src

* Actions to describe

** Knowledge.Link

| Action      | Type    | Required description |
|-------------+---------+----------------------|
| ~:create~   | create  | "Assert a typed relationship between two blocks (e.g. supports, contradicts, depends_on)." |
| ~:destroy~  | destroy | "Remove a previously asserted relationship between two blocks." |

Per-arg / accepted attribute descriptions:
- ~:source_block_id~ — "The block making the assertion."
- ~:target_block_id~ — "The block being related to."
- ~:relation~ — "Relationship kind. See Knowledge.Link.relation_types/0 for the allowed set."

** Knowledge.Citation

| Action          | Type   | Required description |
|-----------------+--------+----------------------|
| ~:for_message~  | read   | "List citations grounding a specific chat message." |
| ~:for_block~    | read   | "List citations involving a specific block (forward and backward)." |
| ~:create~       | create | "Record a citation tying a chat message to a source block." |

Per-arg:
- ~:message_id~ — "Chat message id whose citations to load."
- ~:block_id~ — "Block id whose citation participation to load."

** Knowledge.IngestionJob

Read the existing actions in ~lib/concept/knowledge/ingestion_job.ex~.
Distinguish *user-visible* vs *internal* — internal actions (driven by
the AshOban worker: ~:run~, ~:succeed~, ~:fail~) get NO description.

| Action      | Type   | Describe? | Description |
|-------------+--------+-----------+-------------|
| ~:read~     | read   | YES       | "List ingestion jobs (queued, running, succeeded, failed) for the workspace." |
| ~:enqueue~  | create | YES       | "Enqueue a page for ingestion (chunking + embedding + graph build)." |
| ~:archive~  | update | YES       | "Archive a completed or failed ingestion job from the user-visible list." |
| ~:run~      | update | NO        | (worker-only; leave undescribed) |
| ~:succeed~  | update | NO        | (worker-only) |
| ~:fail~     | update | NO        | (worker-only) |

Per-arg:
- ~:enqueue.page_id~ — "Page to ingest."
- ~:enqueue.workspace_id~ — "Workspace the page belongs to."
- ~:enqueue.op~ — "Operation: :upsert (default) or :delete."
