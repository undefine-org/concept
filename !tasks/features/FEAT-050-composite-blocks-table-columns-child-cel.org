* Intention & Purpose

Creating a Table via the slash menu produces a fully usable n×m table: header row + body cells, each editable like a paragraph block, with Tab/Shift-Tab for cell navigation. Creating a Columns block produces n side-by-side columns, each accepting any block type via drag-into / inline slash. Both compose: a column can contain a table.

* Context & Narrative

Resource model (already in place via FEAT-012 BlockType modules):
- =Concept.Pages.BlockTypes.Table= → parent block; props ~%{rows: int, cols: int}~.
- =Concept.Pages.BlockTypes.TableCell= → child block; ~parent_block_id~ points at Table; ~position~ ordered row-major.
- =Concept.Pages.BlockTypes.Columns= → parent block; props ~%{count: int}~.
- =Concept.Pages.BlockTypes.Column= → child block; ~parent_block_id~ points at Columns.

Missing: the creation Reactor (or composite create action) that fans out the parent + N child blocks atomically; the Lit/HEEx renderers for the parent shells that nest =BlockRender= per cell.

Server: =Concept.Pages.create_table/4= (Reactor) accepting workspace_id, page_id, rows, cols → creates 1 Table parent + rows*cols TableCell children. Returns the parent. Similar for Columns.

Client: 
- ~ora-block~ already renders one block. New ~ora-table~, ~ora-columns~ Lit elements wrap them as containers and lay out children via CSS grid.
- Or simpler: =BlockRender= recurses for parent blocks, rendering a HEEx table/grid directly without bespoke Lit elements. Cleaner; less JS.

Choose HEEx recursion for the parent shell, keep ~ora-block~ as the cell editor.

Slash menu (after BUG-017 lands) selecting Table → opens a small "Rows × Cols" picker before insert. Selecting Columns → "2 / 3 / 4 columns" picker.
