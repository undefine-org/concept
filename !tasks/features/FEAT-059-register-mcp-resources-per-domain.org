* Intention & Purpose

Tools are verbs; resources are nouns. Add an MCP browse surface so the
LLM can navigate Concept like a human reads the sidebar — without having
to invoke a verb to discover what exists.

Use AshAi's ~mcp_resource~ DSL (already supported in 0.6.1, added in
v0.4.0 per the changelog).

* Scope (IN)

** ~Concept.Pages~

#+begin_src elixir
mcp_resources do
  mcp_resource :workspace_tree,
    "concept://workspaces/{workspace_id}/tree",
    Concept.Pages.Page, :list_tree,
    mime_type: "application/json",
    description: "Hierarchical page tree of a workspace."

  mcp_resource :page,
    "concept://pages/{page_id}",
    Concept.Pages.Page, :read,
    mime_type: "application/json",
    description: "A page (loaded with metadata)."

  mcp_resource :page_blocks,
    "concept://pages/{page_id}/blocks",
    Concept.Pages.Block, :list_for_page,
    mime_type: "application/json",
    description: "All blocks on a page, in render order."

  mcp_resource :recent_pages,
    "concept://workspaces/{workspace_id}/recent",
    Concept.Pages.Page, :recent_pages,
    mime_type: "application/json",
    description: "10 most recently updated pages in the workspace."
end
#+end_src

* Implementation notes

** URI templates

Verify AshAi's URI template binding mechanism in ash_ai 0.6.1
(~deps/ash_ai/lib/ash_ai/mcp/server.ex~). The current implementation
expects URI template variables to map to action arguments by name.
If the binding requires explicit option (e.g. ~argument_mapping:~),
add it here.

** Tenancy

The plug (FEAT-058) assigns ~:mcp_tenant~. AshAi.Mcp.Router must propagate
this to ~resources/read~ calls — verify by reading the server code. If
the tenant is *not* automatically applied to resource reads, add explicit
tenant resolution via the same MFA pattern used for tools.

** Pagination

For ~list_tree~ and ~for_conversation~, the action accepts pagination
arguments. AshAi MCP returns ~nextCursor~ when paginated — verify the
action has ~pagination~ enabled. If not, leave it for a follow-up; the
MVP returns up to the action's default ~limit~.

* Edge cases

- ~concept://blocks/{block_id}/citations~: the citations action is
  workspace-tenanted, but the URI doesn't carry workspace. Tenancy must
  come from the plug (FEAT-058). Verify isolation by reading citations
  of a block in workspace B with an ApiKey bound to workspace A — must
  return empty or 404, never block B's data.
- Templates must be exact: ~{workspace_id}~ not ~{ws_id}~ etc. Match the
  argument names on the underlying action.
