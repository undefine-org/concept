** GATE QUESTIONS — answered
- *Did the transformer compose cleanly vs Ash compile ordering?* We did NOT need
  a Spark transformer at all. The keystone is achievable with a plain ~__using__~
  mixin + a described ~discuss~ action that AutoTools already projects. The
  riskiest envisioned piece (DSL-patching ~[:actions]~/~[:relationships]~
  mid-compile) was AVOIDED — ~page_discuss~ is synthesized for free because the
  action carries a description. This is the parity principle paying off exactly
  as designed.
- *Is ~use Concept.Hostable~ on par with BlockType mixins?* Yes — one stanza,
  same ergonomics, ~defoverridable~ callback default. Cleaner, in fact, since no
  transformer is required for Wave 0.

** Decisions / deviations from plan
- DEFERRED the per-host Spark transformer. ~discuss~ lives on the host resource
  as a described action (Page.discuss → page_discuss). When a 2nd/3rd host type
  lands (Record), evaluate whether to DRY via a transformer or a shared
  ~use Concept.Hostable.Discussable~ mixin contributing the action. The data
  model + registry + grounded-scope spine is proven; tool synthesis is free.
- Kept the transitional ~relate_actor(:user)~ on Conversation.create (creator
  still related as user) until the Participant model lands (Wave 1). No 1:1
  reflex was ripped out prematurely — additive, non-breaking.
