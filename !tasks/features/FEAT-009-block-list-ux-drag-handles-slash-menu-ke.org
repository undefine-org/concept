* Initial Message

implement a pixel-perfect version of Notion. Use Ash as much as possible. This plan should cover the MVP version.

* Intention & Purpose
Notion-style block manipulation: hover handles =+= (insert below) and =⋮⋮= (drag/menu); slash =/= menu for type insertion; cross-block keyboard navigation (Up/Down moves focus into prev/next =<ora-block>=, Enter at end of paragraph creates new paragraph below, Backspace at start removes empty block and merges with previous). Drag-and-drop reorder via SortableJS pushing =:reorder_block= and =:reparent_block= server actions.

* Context & Narrative
This is the bulk of "Notion feel". All block-type-agnostic — pulls metadata from =Concept.Pages.BlockTypes.slash_menu_items/0=, so when FEAT-010/012 add types they appear in the slash menu automatically.

* Scope
** In-scope
- =assets/js/components/ora-block-handle.js= Lit element rendered next to each =<ora-block>=: =+= and =⋮⋮= buttons with hover/focus styling; emits =add-below=, =open-menu=, =start-drag=
- =assets/js/components/ora-slash-menu.js= Lit popover positioned below caret; receives item list as attribute; arrow keys + Enter; emits =select={detail: {type}}=
- =assets/js/components/ora-block-actions.js= Lit popover (reuse Petal Dropdown styling): Duplicate (FUP, disabled), Delete, Turn into → submenu of types
- =assets/js/hooks/block_list.js= LV hook on =<ul phx-hook="BlockList">= wraps SortableJS:
  - On drop, push =:reorder_block= or =:reparent_block= with new neighbour ids; server computes fractional position
  - Disable drag while a child block is held in lock by another user
- =assets/js/hooks/block_keyboard.js= attached to =<ora-block>= host (composes with BlockEditor):
  - Intercepts ArrowUp at line 0 → emit =nav-up= → LV moves focus to prev block via =push_event focus_block_id=
  - ArrowDown at last line → =nav-down=
  - Enter at end of paragraph (or any block with no children to consume Enter) → =insert_below_paragraph= server event
  - Backspace at offset 0 of empty block → =delete_block_merge= server event
- =ConceptWeb.PageLive= handlers:
  - =:slash_menu_open= (with caret coords) → push state to client to render =<ora-slash-menu>=
  - =:insert_block= (after_id, type) → =Concept.Pages.create_block!/1=, position computed via FractionalIndex.between
  - =:reorder_block= (block_id, prev_id, next_id) → compute position, =reorder_block!=
  - =:reparent_block= same with new parent
  - =:delete_block_merge= → if block empty, archive and focus previous block; if previous is paragraph, append nothing
  - =:turn_into= (block_id, new_type) → archive old, create new with same content if compatible (text-bearing types interchangeable; otherwise new with default)
- "+" floating action above empty page creates first paragraph

* Edge Cases
- Drag a locked-by-other block: SortableJS disables; server still validates and rejects with no-op.
- Reorder to position before first sibling when first sibling has position "a" (lowest possible): FractionalIndex.before extends with new prefix.
- Slash menu opening near viewport bottom: position above caret if not enough room below (handled by Lit element computing bounding rect).
- Backspace merging: if previous block is non-text (divider, image), don't merge — just delete current empty block.
- Concurrent insert: two users press =+= simultaneously; both succeed; positions distinct via fractional indexing (independent random midpoints).
- Slash with text after =/=: only fires menu when caret is at start of empty paragraph; otherwise =/= is a literal character.
