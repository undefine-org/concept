* Initial Message

how would you integrate Arcana into this, maximally leveraging Ash to make the system as simple as possible?

* Intention & Purpose
Embed AI answers directly inside Concept pages as a first-class block type. The user types `/ai`, picks the new block, enters a prompt, and the block renders a cached answer + sources. A "Refresh" button re-runs the pipeline scoped to the current page subtree.

* Context & Narrative
Concept's block system is extensible: `Concept.Pages.BlockTypes` registry compiles from `:concept, :block_types` config. Each type is a module implementing `Concept.Pages.BlockType` with `type/0`, `default_content/0`, `default_props/0`, `validate_props/1`, `lexical_node/0`, `slash_menu/0`, `container?/0`. Adding `Concept.Pages.BlockTypes.AiAnswer` integrates seamlessly.

Semantics (per design):
- props: `%{"prompt" => string, "scope" => "subtree" | "page" | "workspace", "model" => string | nil}`. Scope default = "subtree" (current page + descendants).
- content: `%{"answer" => markdown, "sources" => [%{block_id, page_id, breadcrumbs, snippet, score}], "ran_at" => iso8601, "model" => "gemini-2.0-flash"}`. Empty until first run.
- Lifecycle: created with empty content + a placeholder shell. User clicks "Generate" (or types and submits the prompt), block transitions to `:running`, streams tokens into a transient assign (live update via PubSub), then on `:knowledge_done` the action `Concept.Pages.Block.evaluate_ai_block` writes content atomically.
- Refresh button re-runs pipeline + replaces content.

* Scope
- IN: `Concept.Pages.BlockTypes.AiAnswer` module; registry registration; new `evaluate_ai_block` Block action; UI render in `BlockRender` (markdown + sources + refresh button + status); slash menu entry under group `:ai`; lock semantics (only block owner can refresh while locked).
- OUT: edit the answer manually, stream INTO the block content as tokens land (just shows full answer once done; transient streaming displayed in UI but content stored only on completion).

* Existing Patterns (MUST follow)
- BlockType behaviour exemplified by `lib/concept/pages/block_types/code.ex` and `paragraph.ex`.
- Block actions live on `Concept.Pages.Block` resource (lib/concept/pages/block.ex). Add `update :evaluate_ai_block` action.
- Slash menu group atoms used: `:basic`, `:media`, `:advanced`, `:hidden`. Add `:ai` group; update `lib/concept/pages/block_types.ex` `slash_menu_items/0` if it filters by group.
- Block render dispatches by `@text_types` list in `block_render.ex`. Add a static branch for `:ai_answer` (it is NOT a Lexical-editable text block).
- Atomic action update: AshPaperTrail records the change.

10. Streaming: while running, UI shows partial text from PubSub; on done, block updates, transient state cleared.

  ```elixir
  defmodule Concept.Pages.BlockTypes.AiAnswer do
    @behaviour Concept.Pages.BlockType
    @scopes ~w(subtree page workspace)
    def type, do: :ai_answer
    def default_content, do: %{}
    def default_props, do: %{"prompt" => "", "scope" => "subtree", "model" => nil}
    def validate_props(%{"scope" => s} = props) when s in @scopes,
      do: validate_prompt(props)
    def validate_props(_), do: {:error, "scope must be one of #{Enum.join(@scopes, ",")}"}
    defp validate_prompt(%{"prompt" => p}) when is_binary(p), do: :ok
    defp validate_prompt(_), do: {:error, "prompt must be a string"}
    def lexical_node, do: "ai-answer"
    def slash_menu, do: %{label: "AI Answer", icon: "✨", keywords: ~w(ai answer ask), group: :ai}
    def container?, do: false
  end
  ```

* Edge Cases
- prompt is empty string: action returns `{:error, :no_prompt}`; UI shows the textarea instead of a refresh button.
- LLM error during evaluation: content remains unchanged; UI shows error toast + retains stale answer if any.
- User navigates away during generation: PubSub messages still flow; on return the block content reflects the latest run (paper_trail captures intermediate runs as separate versions).
- Cycles: AI block scoped `:subtree` can include itself (its prompt becomes context). Mitigation: filter out chunks whose `block_id == self.id` in scope_filter.
- Workspace scope on huge workspaces: top-K = 10 chunks → bounded latency.
- AI block inside a `columns` container: render OK; overflow handled by column width.
