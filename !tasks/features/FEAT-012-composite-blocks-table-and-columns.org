* Initial Message

implement a pixel-perfect version of Notion. Use Ash as much as possible. This plan should cover the MVP version.

* Intention & Purpose
Implement Notion-style =:table= and =:columns= as container BlockType modules that compose existing simple blocks via =parent_block_id=. Use Ash =bulk_create= so inserting a 2-column or 3×3 table is a single transaction. Drag handles work within composite child slots.

* Context & Narrative
This proves the BlockType registry handles composites without changing the Block resource. Tables here are the *layout* table block (rows of cells of paragraphs), not the Notion "database" feature (out of scope, FUP).

* Scope
** In-scope
- =Concept.Pages.BlockTypes.Table= module:
  - =container?/0 → true=
  - =default_props/0 → %{rows: 2, cols: 2, has_header_row: true, column_widths: [200, 200]}=
  - =default_content/0 → %{}= (cells live as child blocks)
  - On =:create_block= for type =:table=, =Concept.Pages.Blocks.create_table_with_cells/2= runs in a Reactor (or Ash multi-action change) that:
    - Creates the parent table block
    - Bulk-creates =rows × cols= paragraph child blocks via =Ash.bulk_create=, each with =parent_block_id=table.id=, type =:table_cell= (which is itself a BlockType wrapping paragraph behavior)
  - Cells use a thin =:table_cell= BlockType (also added in this FEAT) that wraps paragraph; its render_static is =<td>= rather than =<p>=
- =Concept.Pages.BlockTypes.Columns= module:
  - =container?/0 → true=
  - =default_props/0 → %{count: 2, ratios: [0.5, 0.5]}=
  - On creation, bulk-creates =:column= child blocks; each column is itself a container that holds simple blocks
- =Concept.Pages.BlockTypes.TableCell= and =Column= modules (auxiliary container types)
- =assets/js/components/ora-table.js= Lit element: renders =<table>= grid; resize handles between columns adjust =props.column_widths=; Tab/Shift-Tab navigate cells; Enter inside cell breaks line; ArrowRight at end of cell jumps to next cell
- =assets/js/components/ora-columns.js= Lit element: flexbox row; resize divider between columns adjusts ratios; each column slot accepts dropped blocks
- BlockList hook nesting: each container block emits its own =<ul phx-hook="BlockList">= scoped to its children; SortableJS allows cross-list drag for reparenting (already supported in FEAT-009)
- Slash menu groups: Table + Columns appear under =:advanced= group

** Out-of-scope
- Notion databases (table-as-database with property schemas) — FUP

* Existing Patterns
- =Ash.bulk_create=: =Ash.bulk_create(records, Block, :create_block, return_records?: true)=
- =Reactor= for atomic create-with-children
- =@lexical/table= node — NOT used; we use child Block records instead so table cells remain first-class blocks (lockable, paper-trailed). Each cell hosts its own =<ora-block>= editor.

* Tests
File: =test/concept/pages/block_types/table_test.exs=, =columns_test.exs=, =test/concept_web/live/composite_blocks_test.exs=.

Scenarios:
1. =:table= create with rows=2 cols=2 produces 1 parent + 4 cell child blocks in one transaction; failing the bulk insert rolls back the parent.
2. Adding a row → bulk_create N cells with new row index in props (cells are ordered by =position= and assigned =props.row_index=, =props.col_index=).
3. Resizing column updates =table.props.column_widths=.
4. =:columns= create with count=3 produces 1 parent + 3 child column blocks.
5. Dragging a paragraph into a column slot reparents it; =parent_block_id= updates; =workspace_id= preserved.
6. Tab in cell N moves caret to cell N+1; if at last cell of last row, creates new row.
7. Archiving the parent table archives all cells in cascade (AshArchival default does NOT cascade — explicit change implements it).
8. Locking cell A by user A does not lock cell B; user B can edit cell B simultaneously.

* Edge Cases
- Empty row deletion: archiving any cell does not delete the row; row deletion is a separate UI action that archives all cells in that row.
- Adding column to wide table: bulk_create one cell per existing row; positions interleaved via fractional indexing.
- Drag block into column already at capacity: no capacity limit; columns scroll vertically.
- Resize ratio < 10%: clamp to 0.1 minimum; UI snaps.
- Concurrent table rebuild from two users (rare): last-writer wins on column_widths; cells unaffected.
- Locking a row cell during edit doesn't block resize on column (resize updates parent props, separate row).
