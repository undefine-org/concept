* Initial Message

Do a critical review of Arcana's integration into Concept and how you'd make the frontend interesting to use w.r.t those features. Go wild, go free, be thorough

* Intention
Today, any block edit re-embeds the whole page. For long pages (50+ blocks), this is a token+latency cliff. Add a content_hash check that skips embed for unchanged chunks; re-uses prior chunk's vector.

* Trigger
Token-cost telemetry from FEAT-048 shows the cliff; or a workspace with 100+ blocks per page surfaces during dogfood.

* Design substrate (already in place)
- BlockChunker emits chunk text + metadata.block_ids — natural keys for content addressing.
- Arcana likely already supports chunk dedupe; verify.

* Implementation sketch
- Chunker computes ~content_hash = :crypto.hash(:sha256, text)~ per chunk; included in metadata.
- Reactor ~build_chunks~ step queries existing Arcana.Chunk by source_id + content_hash; for matches, marks chunk as ~:reuse~.
- ~upsert_document~ step skips embed call for reuse chunks; copies prior vector.
- Worst case (all chunks changed) is no slower than today.
