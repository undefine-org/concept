* Initial Message

PLAN-005 W2.7.6 Slice E re-run (P3 UX): clicking a button inside `<ora-format-toolbar>.shadowRoot` (Bold/Italic/Link) causes `document.activeElement` to leave the contenteditable → `selectionchange` fires with empty selection → FormatToolbar hook resets `visible` to false → the click's apply-link/toggle-format path runs against a hidden toolbar that may have already cleared state. End behavior currently works because the click handler fires synchronously before the reset, but real users with slower hands or different OS focus models could lose the apply.

* Fix sketch
- In `<ora-format-toolbar>`: prevent focus loss on click via `onPointerDown` `event.preventDefault()` on buttons.
- OR in the FormatToolbar hook: ignore `selectionchange` events for ~150ms after a button click (debounce via setTimeout flag).
- Add vitest scenario that asserts toolbar remains `visible` through a button click.
