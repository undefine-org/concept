* Summary
After BUG-048, six workspace-tenant resources now repeat the same wiring:

#+begin_src elixir
multitenancy do
  strategy :attribute
  attribute :workspace_id
  global? false
end

relationships do
  has_many :workspace_memberships, Concept.Accounts.Membership,
    no_attributes?: true,
    filter: expr(workspace_id == parent(workspace_id))
end

policies do
  bypass actor_attribute_equals(:system?, true) do
    authorize_if always()
  end

  policy action_type(:create) do
    authorize_if Concept.Pages.Checks.WorkspaceMemberCreate
  end

  policy action_type([:read, :update, :destroy]) do
    authorize_if Concept.Pages.Checks.WorkspaceMember
  end
end
#+end_src

Live in: =Block=, =Page=, =Knowledge.Link=, =Knowledge.Citation=,
=Knowledge.TokenLedger=, =Knowledge.IngestionJob=. Every new
workspace-tenanted resource will need the same boilerplate or auth either
over-issues queries (forgetting the FilterCheck) or breaks =:create=
(forgetting =WorkspaceMemberCreate=).

* Goal
Collapse the four-block recipe into a single declaration:

#+begin_src elixir
use Concept.Resources.WorkspaceTenanted
#+end_src

The extension injects:
1. =multitenancy= attribute strategy on =:workspace_id=.
2. The =:workspace_memberships= =has_many= relationship.
3. The two membership policies (create / R-U-D) with system-actor bypass.

Resources retain the freedom to add extra policies (e.g. role gating) by
appending more =policy= blocks; the macro adds to the policies DSL section
rather than replacing it.

* Implementation sketch
- Ash supports composable resource extensions via =Spark.Dsl.Extension=. The
  extension is a tiny module that registers DSL entities through
  =@dsl_patches= or a transformer.
- For pragmatism, a simpler =__using__= macro that emits the boilerplate
  works too, and matches the style of =Concept.AshOban.WorkspaceTenants=
  hoist done in FUP-027. Start there; promote to a Spark extension only if
  the macro produces noisy duplicate-policy warnings.
- Place at =lib/concept/resources/workspace_tenanted.ex=.

* Migration
- Replace the four blocks in each of the six resources with =use=.
- Resources that already split their =:create= policy (e.g. =Citation= where
  =:create= is system-only) keep their override; the macro's =:create=
  policy can be opted out via an option:
  =use Concept.Resources.WorkspaceTenanted, create_policy: :system_only=.
