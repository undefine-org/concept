* Context

** Goal
Ship an MVP of a Notion-inspired collaborative document workspace, Ash-first across the entire stack. Output: visitor signs up → personal workspace auto-created → can build nested page trees, edit pages with 14 block types using a Lexical-powered editor wrapped in Lit web components, format inline marks, drag-reorder blocks, search via Cmd-K, and collaborate in real time with paragraph-level locking + presence avatars. Light theme only, looser fidelity (Notion-inspired, not strictly pixel-exact).

** Non-goals (explicit out-of-scope, deferred to FUP)
- Notion databases (table/board/gallery views with property schemas)
- Comments, mentions, reminders
- Public sharing, ACLs beyond workspace membership, invite-by-email
- File uploads (S3, local disk) — only emoji icons + color covers + URL-paste image/bookmark
- Full-text search across block content (only title ILIKE in MVP)
- Dark mode, mobile responsive, A11y full audit
- Real CRDT (paragraph-level lock instead per AGENTS.md)
- Templates, AI features, third-party integrations
- Cross-block undo/redo (only within Lexical block instance)
- Cell merging, code syntax highlighting, equation editor (renders KaTeX read-only output)

** Settled design decisions
1. *Stack*: Phoenix 1.8 + LiveView 1.0 + Elixir 1.19; Ash 3 + AshPostgres + AshAuthentication + AshAuthenticationPhoenix + AshPhoenix + AshArchival + AshPaperTrail + AshStateMachine + AshOban + AshAdmin; Postgres 16 (with pg_trgm, citext); Tailwind 3 + PetalComponents 2 for chrome; custom Lit 3 web components for editor; Lexical 0.17 (headless) + KaTeX for equations + SortableJS for drag.
2. *Auth*: ash_authentication password strategy (email + password). No magic link, no OAuth.
3. *Workspaces*: multi-workspace from day 1, single auto-created personal per user. Workspace-scoped sharing only — every workspace member sees every page. Switcher UI deferred.
4. *Multitenancy*: Ash =multitenancy strategy: :attribute, attribute: :workspace_id= on Page + Block resources. All reads/writes auto-scoped.
5. *Pages*: nested tree via =parent_page_id=; fractional indexing (LexoRank-like) on =position= so insertion never re-shuffles siblings; soft-delete via AshArchival; version history via AshPaperTrail.
6. *Blocks*: ★ single =Concept.Pages.Block= Ash resource + =Concept.Pages.BlockType= behaviour + explicit-config registry. New type = one module + one config line. State machine for lock states (=:unlocked → :locked=). 14 user-facing types: paragraph, H1/H2/H3, bulleted, numbered, to-do, quote, divider, code, callout, toggle, image (URL), bookmark (URL), equation (KaTeX), plus 4 internal aux types (table, table_cell, columns, column) — registry total 18.
7. *Editor*: one Lexical instance per block (=<ora-block>= Lit element). Cross-block keyboard nav via LV hook (=ArrowUp/Down=, =Enter at end=, =Backspace at start=). Inline marks (bold/italic/underline/strike/code/link) via Lexical native formats + floating Petal-styled toolbar.
8. *Realtime*: paragraph-level lock via state machine; lease TTL 30s refreshed every 15s via heartbeat; AshOban trigger releases stale locks every 10s. Phoenix.PubSub for content broadcast; Phoenix.Presence for avatars on page topic =workspace:#{ws}:page:#{p}:presence=.
9. *Onboarding*: Ash.Reactor =Concept.Accounts.Reactors.Onboarding= replaces =after_action= hook — atomic with compensations.
10. *AshAdmin* mounted at =/admin= behind owner-only plug.
11. *Search*: title-only ILIKE with pg_trgm gin index.

** Architecture map
- =Concept.Accounts= domain → User, Token, Workspace, Membership; AshAuthentication strategies; Onboarding Reactor
- =Concept.Pages= domain → Page (multitenant), Block (multitenant + state machine), BlockType behaviour + BlockTypes registry, FractionalIndex, Lexical helpers
