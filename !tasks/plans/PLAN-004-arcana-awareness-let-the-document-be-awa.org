* Goal
Make Arcana a first-class document primitive in Concept, not a bolted-on chatbot. Every retrieval, answer, citation, and link is a tracked Ash resource. The document becomes self-aware: it indexes itself, references itself, asks itself questions, and surfaces those answers ambient to the act of writing.

* Settled decisions
- *Gemini 3.x family* via ReqLLM "google:..." model strings (May 2026 lineup):
  - Embeddings: ~google:gemini-embedding-2~ (multimodal — unlocks FUP-012 image search via the same endpoint, no separate CLIP embedder needed).
  - ~:fast~ profile: ~google:gemini-3.1-flash-lite~ (stable) — rewrite, intent classification, grounding.
  - ~:default~ profile: ~google:gemini-3-flash~ (preview) — default chat answers.
  - ~:thorough~ profile: ~google:gemini-3.1-pro~ (preview) — outline drafting, loop.
  Stable fallbacks: ~gemini-2.5-flash~ / ~gemini-2.5-pro~ / ~gemini-2.5-flash-lite~ if previews unavailable. Single ~GOOGLE_API_KEY~ for v1 (per-workspace keys = FUP-004).
- *Adopt AshAI* (~ash_ai ~> 0.6.1~) alongside ReqLLM (AshAI uses ReqLLM internally). Maps onto our design:
  - ~mix ash_ai.gen.chat --live~ generates ~Knowledge.Conversation~ + ~Knowledge.Message~ resources with state machines, AshOban backing, and PubSub streaming → replaces the custom ~Knowledge.Query~ resource I originally proposed (FEAT-033 narrows to "extend generated Message with citations + search_trace").
  - ~AshAi.ToolLoop.stream/2~ replaces the custom ~Concept.Knowledge.Answerers.StreamingGemini~ module entirely (uses ReqLLM streaming + Phoenix.PubSub natively).
  - Domain-level ~tools do ... end~ Spark block exposes Knowledge.search / Knowledge.ask / Pages.create_page / Knowledge.Link.create / Pages.summarize_subtree as LLM tools → FUP-007 intent bar becomes ~AshAi.ToolLoop.run~ with no custom routing.
  - ~AshAi.Actions.Prompt~ for prompt-backed Ash actions → AI block evaluation collapses to one action declaration.
  - MCP server (~mix ash_ai.gen.mcp~) exposes Concept's tools to Claude Desktop / Cursor / etc. — promoted to FEAT-049.
  - AshAI's ~on_tool_start~/~on_tool_end~ callbacks substrate the Why-this-answer panel (FEAT-044).
- *Maximize Ash modeling*. Keep ad-hoc-promoting resources: ~Knowledge.IngestionJob~ (retrieval-side; AshAI doesn't cover ingestion), ~Knowledge.Citation~ (belongs_to AshAI Message; UI-shaped deep-link metadata), ~Knowledge.Link~ (user-authored graph edges; not LLM-mediated). AshAI provides Conversation + Message for free.
- *Spark DSL* for ~Concept.Knowledge.Profiles~ — declarative bundle of (rewrite/search/rerank/ground) + (Gemini model) + (which tools to expose). Compile-time validation of model strings against the Gemini family regex. Introspectable. Picked per call via ~profile: :fast | :thorough | :outline | :contradict | :intent~ atom on Knowledge.ask.
- *Reactor* drives the ingest pipeline. Step-local retry (only embed retries on 429). Compensation undoes orphan documents on failure. Future FUP-010 dirty-hash slots in as a new step with no structure change.
- *AshOban* drives all background work: IngestionJob, weekly community rebuild, daily token aggregation.
- *AshStateMachine* on IngestionJob + generated Message lifecycles.
- *AshPaperTrail* on Knowledge.Link (edit history of authored edges).
- *AshArchival* on IngestionJob + Conversation + Message (soft-delete old rows).
- Streaming: AshAI's ~ToolLoop.stream/2~ broadcasts tokens via Phoenix.PubSub on ~conversation:<id>~ topic out of the box. Our LiveComponents subscribe.
- Tenancy: workspace_id attribute multitenant on every Knowledge resource. System actor bypass via ~bypass actor_attribute_equals(:system?, true)~ uniformly.
- AI block stores ~{message_id, model, ran_at}~ in content; the durable answer lives on the AshAI Message row. Unlocks Why-this-answer + lineage from a single source of truth.
- User-authored graph edges (~Knowledge.Link~) write into ~Arcana.Graph.Relationship~ via after_action change (idempotent upsert). Provides substrate for the Workspace Graph view.
- Wild moves filed as FUPs; substrate prepared so each fits naturally without redesign.
