* Initial Message

how would you integrate Arcana into this, maximally leveraging Ash to make the system as simple as possible?

* Context
Concept is a Phoenix 1.8 + Ash 3 Notion-like app (Workspace → Page tree → Block tree, Lexical content, AshOban + ash_authentication + ash_paper_trail + ash_archival). We add Arcana v2.0.0 to bring RAG + structural GraphRAG to the workspace.

** Settled decisions
- Surfaces (4): /admin/arcana Dashboard (owners), workspace Q&A chat panel sidebar, command palette semantic search, AI answer block_type.
- Mode: Arcana.Pipeline (rewrite → fusion search → answer with streaming).
- Tenancy: one Arcana collection per workspace ("workspace:<uuid>"); enforced at Knowledge action layer (not Arcana itself).
- Ash wrapping: new Concept.Knowledge Ash domain with generic actions (`ingest_page`, `delete_page`, `search`, `ask`, `rebuild_workspace_graph`, `rebuild_communities`, `evaluate_ai_block`); Arcana keeps its own Ecto schemas.
- Ingestion: Ash.Notifier on Page+Block → AshOban worker (debounced via Oban unique replace_at semantics, scheduled +2s).
- Doc granularity: one Arcana.Document per Page; chunked by Concept.Knowledge.BlockChunker (Lexical→markdown, block-aware, sibling coalescing).
- Providers: Gemini text-embedding-004 (768d) + gemini-2.0-flash chat via ReqLLM; hard-fail without GOOGLE_API_KEY.
- Streaming: Arcana.Pipeline.answer(stream: true) → Stream.each broadcasts to "knowledge:answer:<uuid>" PubSub topic; LiveView consumes via stream(:messages).
- Citations: each chunk metadata carries block_id+page_id+breadcrumbs; UI renders clickable card linking to /w/:slug/p/:page#block-:id; new .PageScroll colocated hook scrolls + flashes.
- GraphRAG: structural graph built from Concept's tree (page, block, parent_of, contains, has_type) written DIRECTLY to Arcana.Graph.Entity/Relationship schemas (no public API; pinned ~> 2.0; schema regression test). Community summaries via weekly AshOban + manual rebuild button.
- AI block: cached answer in block.content; Refresh button re-runs scoped pipeline (default scope=subtree).
- Telemetry: arcana.* + concept.knowledge.* events surfaced in /dev/dashboard.

  2. Wait for AshOban ingest queue to drain. Verify `arcana_chunks` rows present, vector dim = 768.
  3. Verify `arcana_entities` and `arcana_relationships` rows include page+block nodes with correct parent edges.
