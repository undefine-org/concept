* PLAN AI Block Declarative Refactor

** Goal

Make block-type wiring (Ash action ↔ LV event ↔ JS hook ↔ Lit component) a *compile-time invariant* owned by the block-type module. Eliminate the 4 hand-rolled hops that left ~:ai_answer~ non-functional.

Bug under the hood:
- ~static_block/1~ in ~ConceptWeb.BlockRender~ emits raw ~<ora-ai-block>~ with no ~phx-hook="AIBlock"~ → ~AIBlock~ JS hook never mounts.
- ~AIBlock~ hook uses ~pushEventTo("#ai-<id>", …)~ targeting a custom element (not a LiveComponent) → push misroutes.
- ~PageEditorLive~ defines no ~handle_event("evaluate_ai", …)~ → would crash even if routed.
- Streaming tokens are subscribed in the spawned ~EvaluateAi~ task process, not the LV → no live updates; only final ~block_updated~ broadcast.

** Non-goals (this iteration)

- Refactoring text blocks / composites away from current ~text_block~/~composite_*~ paths.
- Live token-streaming UI. Rely on existing ~block_updated~ PubSub broadcast → re-render to ~answered~.
- Generic JS hook adoption for non-AI interactive blocks. Macro + hook + AI conversion only; other blocks adopt opportunistically.

** Architecture

#+begin_example
BlockType (behaviour) ── render/1, live_component?/0
   ├── Static (use Macro)      → render only
   └── Interactive (use Macro)
        ├── declares ash_actions: …
        ├── generates handle_event/3 + Phoenix.LiveComponent boilerplate
        ├── compile-time AST check: data-events ⊆ ash_actions keys
        └── render/1 must include phx-hook="OraBlock"

dispatcher ConceptWeb.BlockRender.block/1
   ├── text / composite paths unchanged
   ├── live_component?  → <.live_component module={mod} id=… block=… />
   └── else             → mod.render(assigns)

JS  OraBlock hook ── reads data-events, forwards ora-<verb> → pushEventTo(this.el, verb, …)
                  ← receives ora:token push, forwards to first child element
#+end_example

**** T-5.1 — Create OraBlock hook

~assets/js/hooks/ora_block.js~:
#+begin_src js
export const OraBlock = {
  mounted() {
    const verbs = (this.el.dataset.events || "").trim().split(/\s+/).filter(Boolean)
    const blockId = this.el.dataset.blockId
    this._listeners = verbs.map(verb => {
      const fn = (e) => this.pushEventTo(this.el, verb, { block_id: blockId, ...(e.detail || {}) })
      this.el.addEventListener(`ora-${verb}`, fn)
      return [verb, fn]
    })
    this.handleEvent("ora:token", (p) => {
      if (p.block_id !== blockId) return
      this.el.firstElementChild?.appendToken?.(p.token)
    })
  },
  destroyed() {
    this._listeners?.forEach(([v, fn]) => this.el.removeEventListener(`ora-${v}`, fn))
  }
}
export default OraBlock
#+end_src
