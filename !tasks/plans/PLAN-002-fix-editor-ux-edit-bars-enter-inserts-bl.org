* Context

Three independently-reproducible UX bugs in the page editor at =http://localhost:4279/w/:slug/p/:id=:

1. *Edit bars not showing* — Lit components ~<ora-block-handle>~ (hover "+/⋮⋮") and ~<ora-format-toolbar>~ (selection bubble) are registered as custom elements but never rendered in any HEEx template. ([[id:BUG-001-edit-bars-block-handle-format-toolbar-do]])

2. *Enter doesn't insert a new block, page appears to reload* — the ~BlockKeyboard~ JS hook is exported but never attached to a DOM element, the server has no ~handle_event~ for ~insert_paragraph_below~ / ~nav_block~ / ~delete_block_merge~, and ~<ora-block>~ does not ~preventDefault~ on Enter so Lexical inserts an internal paragraph. The user sees no visible new block; intermittent crashes plus the duplicate-id risk in ~add_first_block~ produce the "page reloads" symptom. ([[id:BUG-002-enter-inside-a-block-does-not-insert-a-n]])

3. *Title not editable on click* — =WorkspaceLive= renders a plain ~<h1>~ instead of using the existing ~ConceptWeb.Components.PageHeader~ live component (which already implements emoji/cover/contenteditable title). The Ash PubSub handler in =WorkspaceLive= also matches the wrong message shape so renames never refresh the sidebar or browser tab title. ([[id:BUG-003-page-title-is-not-editable-rename-does-n]])

Approach: each bug is a pure wiring/refactor with high information-density. Prefer reusing existing modules (~PageHeader~ live component, ~FractionalIndex~, ~Pages.create_block~, ~BlockEditor~ hook) over creating new ones. Tests live under =test/concept_web/live/= using the auth/setup pattern from =command_palette_test.exs=.
