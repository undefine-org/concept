* Observation (2026-05-29, QoL review M2)

=lib/concept_web/components/page_header.ex= renders the emoji picker and cover
color popovers gated on =@show_emoji_picker= / =@show_cover_picker=. They open
via =toggle_emoji_picker= / =toggle_cover_picker=. There is *no outside-click
(click-away) dismissal* — once open, a popover stays open until the user clicks
the same toggle button again or completes a selection. Clicking elsewhere on
the page leaves a stranded floating popover. Standard popovers dismiss on
outside click; users expect this.

* Decision

Use Phoenix.LiveView.JS click-away via a backdrop layer OR a
=phx-click-away= binding on the popover container that pushes the existing
=toggle_*= (or a dedicated =close_*=) event to @myself. Prefer
=phx-click-away= on the popover element targeting the component — minimal,
no inline JS, mirrors the link-modal sibling-layer fix only if needed.

* Scope
- lib/concept_web/components/page_header.ex — add phx-click-away to both
  popover containers; add close_emoji_picker/close_cover_picker handlers (or
  reuse toggles).
- Test: open picker via toggle event, send click-away, assert popover gone.

* Acceptance
