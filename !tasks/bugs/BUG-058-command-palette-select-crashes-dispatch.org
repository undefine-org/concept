* Intention & Purpose

Selecting a search result in the Cmd-K palette (click or Enter) navigates to that page/block. This is the single most-used palette action; it must never crash.

* Symptom (confirmed live, 2026-05-29)

In =lib/concept_web/live/command_palette_live.ex=, =dispatch_selected/1= and the semantic =title_page_ids= read =assign_async= results one level too shallow. The async assigns wrap their payload in a named map:

- =title_results.result= = =%{title_results: [%Page{}, ...]}=
- =semantic_results.result= = =%{semantic_results: [%Hit{}, ...]}=

=render/1= reads them correctly (=@title_results.result.title_results=). But =dispatch_selected/1= reads =.result= directly:

- Title branch: =socket.assigns.title_results.result |> Enum.take(4) |> Enum.at(page_idx)= → enumerating a 1-key map yields ={:title_results, [...]}= (a tuple) → =page.id= raises =KeyError= (tuple has no =.id=).
- Semantic =title_page_ids=: =Enum.map(socket.assigns.title_results.result, & &1.id)= → same tuple → KeyError.
- Semantic hits: =socket.assigns.semantic_results.result |> Enum.uniq_by(& &1.page_id)= → KeyError.

Actions (New page / Sign out) sit below =action_count= and don't touch =.result=, so they work — masking the bug. Any Pages or Semantic selection crashes the LiveComponent → palette dies, reconnect flicker.

* Root cause

Inconsistent access between =render/1= (correct, =.result.title_results=) and =dispatch_selected/1= (wrong, =.result=). The fix is to read =.result.title_results= / =.result.semantic_results= in dispatch, mirroring render. Prefer extracting a private accessor (e.g. =title_pages(assigns)= / =semantic_pages(assigns)=) used by BOTH render and dispatch so they can never diverge again.

* Scope

- IN: =command_palette_live.ex= — fix the 3 mis-reads in =dispatch_selected/1= (title nav, semantic title_page_ids, semantic hits).
- IN: extract shared accessors so render + dispatch use one source.
- IN: ExUnit regression: select a title result → asserts ={:palette_navigate, page_id}= sent; select a semantic result → ={:palette_navigate, page_id, block_id}=.
- OUT: palette visual redesign, a11y (separate).

* Existing Patterns
