* Initial Message

W2.5 specialist reviewer flagged (P1, conf 0.95): =assets/js/hooks/slash_menu.js= ~_handleSelectItem~ never actually removes the leading "/" from the Lexical editor before inserting the new block. Root causes:
1. ~$getSelection~ is not imported into the module; the runtime guard ~typeof $getSelection === "function"~ is always false; the Lexical-side deletion branch is never reached.
2. The saved ~_triggerRange~ is a /collapsed/ DOM range cloned right after the "/" was typed \u2014 ~range.cloneRange()~ of a selection whose ~startOffset~ already moved past the "/". Even if step 1 were fixed, ~removeText()~ on the resulting collapsed Lexical selection would be a no-op.

Result: after the user picks a menu item, the new block appears below but the source block still contains the stray "/" (plus any filter chars typed). Violates the BUG-017 contract bullet: "restore trigger range \u2192 Lexical ~editor.update(() => $getSelection().removeText())~ to drop the slash + filter chars".

W2.5 also flagged (P2): =assets/js/hooks/format_toolbar.js= listens for a ~cancel-link~ CustomEvent that no component dispatches. Dead listener. Bundle the fix here since both touch slash/format flow.

* Intention & Purpose

After picking a slash-menu item, the source block's content reflects /only/ what the user typed before "/" \u2014 the slash and filter chars are deleted from Lexical. Vitest asserts deletion explicitly so this regression is caught.

Format-toolbar's ~cancel-link~ listener is either removed (current cleanest path) or paired with a real dispatcher in =<ora-link-editor>= (if we want explicit cancel behavior).

* Context & Narrative

Lexical 0.17 selection API needs to be /used inside =editor.update(() => \u2026)= callback/, where ~$getSelection()~ is callable as a module helper from ~lexical~. The current code tries to read DOM selection then map it; for our case the simpler path is: instead of replaying the DOM range, store the /pre-slash content length/ at trigger time, then on select do ~editor.update(() => {root.getLastDescendant().select(preLen, preLen + slashChars)})~ then ~$getSelection().removeText()~.

For ~cancel-link~: =<ora-link-editor>= has its own ~visible~ reactive prop; Escape inside the editor sets ~this.visible = false~ via internal Lit binding. We do NOT need an external cancel handler. Remove the listener.
