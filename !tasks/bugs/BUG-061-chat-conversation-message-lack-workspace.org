* Intention & Purpose

~Concept.Knowledge.Chat.Conversation~ and ~.Message~ are plain ~Ash.Resource~
(no ~workspace_id~, not ~WorkspaceTenanted~). The ~:respond~ AshOban trigger
carries no tenant. So inside ~Respond.do_stream/2~, ~context.tenant~ is nil and
the workspace is unknown — retrieval (collection selection) and Citation
creation (workspace_id is ~allow_nil?: false~) are both impossible.

Prerequisite for BUG-052 (chat RAG wiring). Discovered while implementing it.

* Implementation
1. Migration: add ~workspace_id :uuid~ to ~conversations~ and ~messages~ (backfill
   nullable, then enforce on new rows).
2. Conversation: ~use Concept.Resources.WorkspaceTenanted~; ~:create~ accepts a
   ~workspace_id~ argument; set_attribute. Keep ~relate_actor(:user)~.
3. Message: ~use WorkspaceTenanted~; on ~:create~ derive workspace_id from the
   parent conversation. ~:respond~ trigger: ~use_tenant_from_record? true~ +
   ~list_tenants Concept.AshOban.WorkspaceTenants~ + system actor persister, so
   ~context.tenant~ is populated in ~Respond~.
4. Thread the current workspace from the LiveView chat surfaces into
   conversation creation.
