* Initial Message

PLAN-005 W5.5 Slice M (FEAT-050 retrospective): table creation works — picker fires `insert_composite_below`, 4 cells persist server-side — but in browser each `<td>` contains only an empty `<div class="block-anchor scroll-mt-20">` with NO `<ora-block>` editor inside. Clicking a cell doesn't focus an editor; typing has no effect. Tab navigation between cells can't be tested.

Root cause: BlockRender's text-block branch only renders `<ora-block>` for types in `@text_types ~w(paragraph heading_1 ... toggle)`. `:table_cell` and `:column` aren't in that list, so cells fall through to the `static_block` fallback which returns `raw("")`.

FEAT-050 commit (d65d131) added the composite parent rendering but not the leaf-cell editor mounting.

* Fix sketch
- Add `:table_cell` and `:column` to `@text_types` in `lib/concept_web/components/block_render.ex` — they should render as `<ora-block>` editors just like paragraphs.
- Confirm cells get full keyboard treatment (Enter, ArrowUp/Down, slash menu) within their cell scope.
- Tab handling for cell-to-cell nav already exists in `BlockEditor` hook (FEAT-050 task added it); verify it activates inside cells now that they're editable.

* TDD evidence
- Extend `test/concept_web/live/composite_blocks_test.exs`: after creating a 2x3 table, assert rendered HTML contains 6 `<ora-block block-type="table_cell">` elements (not empty divs).
