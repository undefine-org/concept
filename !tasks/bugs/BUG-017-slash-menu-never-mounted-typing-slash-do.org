* Initial Message

Frontend audit (2026-05-16). =assets/js/components/ora-slash-menu.js= is a 227-line Lit element with keyboard nav, group filtering, FALLBACK_ITEMS containing all 19 block types. =assets/js/components/index.js= side-effect-imports it (registers the custom element). * Intention & Purpose

Typing "/" in any focused block opens the slash menu anchored at the caret. Arrow keys navigate items; Enter selects; Escape closes. Selecting an item dispatches a server event ~insert_block_below~ with the chosen type, persisting via =Concept.Pages.create_block/N=, then focuses the new block.

* Context & Narrative

State machine:
- Per-page: at most one slash menu open at a time.
- Trigger: ~beforeinput~ / ~input~ event on the focused =<ora-block>= where the inserted text is "/" and the prior character is whitespace or start-of-line.
- Position: anchor rect from current selection.
- Filter: subsequent characters typed are forwarded as filter; backspacing past the "/" closes.
- Selection: Enter/Tab/Click → dispatch CustomEvent ~select-item~ with ~{type: "heading_1"}~.

Server side: handler ~insert_block_below~ already exists in =page_editor_live.ex= for paragraph (created by PLAN-002). Extend to accept ~type~ param. Reuse =FractionalIndex.between/2= for position between the source block and its next sibling.

The Lit element's existing API exposes ~items~ attribute (JSON) for dynamic block types. FALLBACK_ITEMS is the default. We can pass server-driven items later (Arcana ai_answer etc.) — out of scope here.

* Scope

- IN: a =assets/js/hooks/slash_menu.js= hook owning the open/close + filter + dispatch lifecycle. Mounted on a page-level host that contains =<ora-slash-menu>=.
- IN: hook listens for ~input~ events bubbling from =<ora-block>= elements; detects "/" at start-of-token; reveals menu at caret rect.
- IN: when item selected → ~pushEvent("insert_block_below", {block_id, type})~; on server-confirmed ~focus_block_caret~ for new id, menu closes.
- IN: server: extend ~insert_block_below~ handler in =page_editor_live.ex= to accept ~type~ param; default ~paragraph~ for backward compat.
- IN: server: ~delete_slash_prefix~ — when a slash-menu item is selected, the host block's content still contains the "/" + filter chars. Server should pop them. Implement client-side by storing the trigger range on open and replacing it with empty on select.
- IN: register hook in =assets/js/hooks/index.js=; render host in =page_editor_live.ex=.
- IN: vitest interaction + LV ExUnit.
- OUT: server-driven items (Arcana / per-workspace types) — substrate ready via ~items~ attribute.
- OUT: nested menus, drag-into-from-menu.
