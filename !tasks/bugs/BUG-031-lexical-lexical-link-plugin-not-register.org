* Initial Message

PLAN-005 W2.6 slice E (browser verification, 2026-05-16) surfaced: the link UI works visually \u2014 toolbar Link button shows =<ora-link-editor>= correctly \u2014 but applying a URL does nothing. Root cause: =assets/js/lexical/registry.js= does not import =LinkNode= from ~@lexical/link~ and does not register a handler for ~TOGGLE_LINK_COMMAND~. The toolbar code in =assets/js/lexical/commands.js= dispatches the command, but Lexical drops it because there's no registered handler and the LinkNode constructor is unknown to the editor.

BUG-016 (DONE, commits 2ce65b9 + ee9559b) wired the toolbar UI + the ~setLink~ command-dispatcher wrapper, but the underlying Lexical config was never extended. Unit tests only assert ~editor.dispatchCommand~ was called with the right args (spy-on-callee), not that the editor state actually changed (the very weakness W2.5 specialist test-depth reviewer flagged as P3 \u2014 the missing =commands.test.js= would have caught this).

* Intention & Purpose

Selecting text + clicking Link in the toolbar + entering a URL wraps the selection in a real ~LinkNode~ that serializes to ~<a href="\u2026">~ on both client render and server ~Concept.Lexical.to_html~. Persists across save+reload. Removal (empty URL) unwraps.

* Context & Narrative

@lexical/link 0.17 API:
- Add ~LinkNode~ to the editor's ~nodes:~ array at editor creation.
- Register a transformer via ~registerLinkPlugin~ or manually subscribe to ~TOGGLE_LINK_COMMAND~ inside the editor's ~register~ callback, calling ~$toggleLink(url)~ inside ~editor.update~.

The standard pattern (mirrors how rich-text plugin is wired in our codebase):
#+begin_src js
import { LinkNode } from "@lexical/link";
import { $toggleLink, TOGGLE_LINK_COMMAND } from "@lexical/link";

// at editor creation:
const editor = createEditor({ nodes: [..., LinkNode], ... });

// at editor mount (inside registry registration):
editor.registerCommand(TOGGLE_LINK_COMMAND, (url) => {
  $toggleLink(url);  // inside editor.update
  return true;
}, COMMAND_PRIORITY_LOW);
#+end_src
