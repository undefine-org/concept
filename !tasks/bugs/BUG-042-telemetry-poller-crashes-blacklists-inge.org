* Symptom

On `mix phx.server` boot (and any time `Concept.Repo` is briefly down/restarting), telemetry_poller logs:

#+begin_example
[error] Error when calling MFA defined by measurement: Concept.Knowledge :report_ingestion_queue_depth []
Class=:error
Reason=%RuntimeError{message: "could not lookup Ecto repo Concept.Repo because it was not started or it does not exist"}
  Ecto.Repo.Registry.lookup/1
  Concept.Repo.aggregate/3
  Concept.Knowledge.report_ingestion_queue_depth/0
  :telemetry_poller.make_measurement/1
#+end_example

* Root cause

`lib/concept/application.ex` starts `ConceptWeb.Telemetry` (which boots `:telemetry_poller`) *before* `Concept.Repo`. The poller's periodic measurement `{Concept.Knowledge, :report_ingestion_queue_depth, []}` calls `Concept.Repo.aggregate/3`, which requires the Repo to be in `Ecto.Repo.Registry`. If the poller ticks while Repo is not registered (slow boot, Repo crash/restart, DB unreachable), the measurement raises.

`telemetry_poller` filters misbehaving measurements via `make_measurements_and_filter_misbehaving/1` — after one raise, the measurement is *dropped from future ticks*. ∴ `last_value("concept.knowledge.ingestion_job.queue.depth")` stops updating until app restart. That's the real damage; the log line is the cosmetic part.

* Fix (option A from analysis)

Make `Concept.Knowledge.report_ingestion_queue_depth/0` defensive:

- short-circuit when `Process.whereis(Concept.Repo)` is nil (no-emit, return :ok)
- rescue any unexpected error and return :ok so the poller never blacklists the measurement

Rejected alternatives:
- Reordering supervision (Repo before Telemetry) — Phoenix defaults to Telemetry-first deliberately so it observes Repo boot events. Don't fight the convention.
- Delay first tick — `:telemetry_poller` exposes no initial-delay knob.

* Scope

- =lib/concept/knowledge.ex= — wrap `report_ingestion_queue_depth/0` body in `if Process.whereis(Concept.Repo) do ... end` with a `rescue _ -> :ok` clause.

* Acceptance

- Booting with Repo intentionally delayed (or stopped) no longer prints the error.
- After Repo recovers, queue-depth metric resumes updating on the next 10s tick.
- `mix precommit` passes.
