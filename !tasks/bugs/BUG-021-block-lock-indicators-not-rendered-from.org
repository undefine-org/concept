* Initial Message

Frontend audit (2026-05-16). =assets/css/app.css= has rules keyed on ~[data-locked-by]~ for a colored left-edge indicator, but no code path writes the attribute. =assets/js/hooks/block_editor.js= handles ~lock_granted~/~lock_denied~ server-pushed events but only as a self-flag (the block being edited); other clients of the same page see no indication that a peer is editing a block. PLAN-001 FEAT-015 claimed "Lock indicator on blocks" + "color hashing helper" done.

* Intention & Purpose

When user A starts editing block X, all other users on the same page see block X with a colored left-bar (the color hashed from user A's id). When A blurs or the lock TTL expires, the bar disappears within ~1s for everyone. Use existing PubSub presence channel + the existing ~ConceptWeb.Colors.for_user_id/1~ helper.

* Context & Narrative

=lib/concept_web/presence.ex= — Phoenix.Presence with topic ~workspace:#{ws.id}:page:#{page.id}:presence~. Metadata already carries ~user_id~ + (potentially) ~locked_block_id~ on focus events.

Today =PageEditorLive= handles ~lock_granted~ from the server but doesn't push a presence diff carrying ~locked_block_id~. The other clients on the same page therefore never learn what block is locked.

Approach: when ~focus_block~ event arrives at the server, =Pages.acquire_lock= currently grants/denies — extend to also update presence metadata for the current LV's tracker entry with ~%{locked_block_id: block_id}~. Sibling LVs' ~handle_info({:DOWN | "presence_diff"}, …)~ rebuilds a map ~%{block_id => %{user_id, color}}~ and re-streams blocks with the new ~locked_by~ assign. =BlockRender= picks the assign up and emits ~data-locked-by~ + ~data-lock-color~.

* Scope

- IN: extend presence metadata schema with ~:locked_block_id~ + ~:color~.
- IN: =PageEditorLive= ~handle_event("focus_block", …)~ now updates presence track entry.
- IN: =PageEditorLive= ~handle_info("presence_diff", …)~ builds ~@locked_blocks~ assign mapping block_id → %{user_id, color}.
- IN: =BlockRender= reads ~@locked_blocks~ (passed in via assigns) and emits ~data-locked-by~ + inline style for the bar color on the ~.ora-block-row~.
- IN: CSS in =app.css= confirms the visual (already partially there).
- IN: ExUnit: two simulated LVs on same page → focus block in one → other observes ~data-locked-by~ attribute.

- OUT: avatar bubbles on individual blocks (separate visual treatment).
- OUT: lock TTL change visualization (countdown).
