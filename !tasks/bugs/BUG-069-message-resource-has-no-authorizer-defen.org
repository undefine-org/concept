* Problem
Message declares data_layer/extensions/notifiers but NO authorizers:
[Ash.Policy.Authorizer] and no policies block — the only workspace-tenanted chat
resource without one (Conversation adds one even as plain Ash.Resource;
Participant uses WorkspaceTenanted). Cross-tenant isolation is NOT currently
broken: multitenancy global? false forces+filters tenant, and the only entry
points (MCP plug verifies membership/binds API key to workspace; LiveView
derives tenant from Workspace.for_user) never hand the actor a foreign tenant.
BUT: (1) message reads apply no participant gating — any actor with the tenant
reads every conversation's messages, defeating the participant-gating
Conversation.read added for private conversations; (2) no defense-in-depth — if
any future call path sets tenant from untrusted input, message content
(the sensitive payload) is fully exposed.

* Fix
Add authorizers: [Ash.Policy.Authorizer] + policy floor mirroring Conversation
(system bypass; read = participant OR WorkspaceMember; create =
WorkspaceMemberCreate), OR `use Concept.Resources.WorkspaceTenanted`. Verify
blast radius (upsert_response, Oban respond, EvaluateAi all authorize?:false —
should be unaffected by the bypass).

* Acceptance
A workspace member who is NOT a participant of a private conversation cannot
read its messages; existing internal/system paths still green.
