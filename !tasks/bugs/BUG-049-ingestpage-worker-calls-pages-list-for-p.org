WHAT HAPPENED???? Please find the structural root causes and explain them to me.

* Summary
=Concept.Knowledge.Workers.IngestPage.perform/1= calls
=Pages.list_for_page(page_id: page_id, actor: actor, tenant: workspace_id)=
but the code interface is defined as =define :list_for_page, args: [:page_id]=,
i.e. =list_for_page(page_id, opts)=. The keyword list lands in the =:page_id=
argument slot and Ash rejects the entire call with
=Ash.Error.Query.InvalidArgument{field: :page_id, …}=.

* Impact
- =Concept.Pages.Notifiers.KnowledgeReindex= enqueues an =IngestPage= job on
  every =:create_block / :update_content / :update_props / :archive / :rename=
  (etc.) notification.
- Every job throws → Oban retries 3× → dead-lettered.
- No page content ever lands in the Arcana RAG collection.
- Downstream AI Answer blocks (=evaluate_ai=) have no context, so
  =wait_for_completion/4= can sit until its 5-minute timeout and the block UI
  never updates its content.

* Root cause
Signature mismatch between the worker and the code interface. The keyword-list
call style was likely copy-pasted from an =Ash.Query=-based helper. Other
callers (=PageEditorLive=, =mix concept.demo=) use the correct positional form:
=Pages.list_for_page(page_id, actor: user, tenant: ws_id)=.

* Fix
=lib/concept/knowledge/workers/ingest_page.ex:22=
#+begin_src elixir
{:ok, blocks} <- Pages.list_for_page(page_id, actor: actor, tenant: workspace_id)
#+end_src
