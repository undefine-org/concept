* Initial Message

Frontend audit (2026-05-16). The "Untitled" placeholder relies on CSS ~.ora-page-title:empty::before { content: attr(data-placeholder); }~. But =ContentEditable= hook saves ~this.el.innerText~ verbatim into ~save_title~. If the user clears the title and then a stray space-key or autofill leaves a single ~" "~ in the element, the title is non-empty (one whitespace char), the ~:empty~ pseudo doesn't match, the placeholder disappears, and the user sees a header with no title and no placeholder.

Same hazard on initial render: if =Page= has ~title: " "~ from a corrupt save, the placeholder is invisible.

* Intention & Purpose

Trim whitespace at the boundaries. Server normalizes incoming titles to ~String.trim/1~ then either ~""~ (which uses placeholder) or a non-empty string. JS hook never sends pure-whitespace values. UI never shows a header with neither title nor placeholder.

* Context & Narrative

Server normalization lives in the =rename_page= action of =Concept.Pages.Page=. Already accepts a string. Add an Ash change ~trim_title~ in =lib/concept/pages/changes/= (mirror existing changes layout).

JS hook trim is cosmetic — server is source of truth — but reduces noise. Do both.

defmodule Concept.Pages.Changes.TrimTitle do
  use Ash.Resource.Change
  @impl true
  def change(cs, _, _) do
    case Ash.Changeset.get_attribute(cs, :title) do
      nil -> cs
      title when is_binary(title) -> Ash.Changeset.change_attribute(cs, :title, String.trim(title))
      _ -> cs
    end
  end
end
#+end_src
