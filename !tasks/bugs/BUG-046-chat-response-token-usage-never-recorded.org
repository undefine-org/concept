* Summary
Audit columns =prompt_tokens=, =completion_tokens=, =grounding_score= on
=Concept.Knowledge.Chat.Message= are never populated for agent responses. Even when
BUG-A is fixed (=extract_usage_metadata/1= stops crashing), it can return nothing
useful — =AshAi.ToolLoop.Result= (ash_ai 0.6.1) doesn't carry usage data.

* Root cause
=AshAi.ToolLoop= consumes the =ReqLLM.StreamResponse= stream with
=Enum.to_list/1= and discards everything except =:content= chunks. =:meta= chunks
carry usage in their =metadata= map but are filtered out by =content_events/1=
in =deps/ash_ai/lib/ash_ai/tool_loop.ex:138=. The =Result= struct only exposes
=[messages, final_text, iterations, tool_calls_made]=.

* Fix (chosen path)
Inject a wrapper =req_llm= module via the existing =:req_llm= option on
=AshAi.ToolLoop.stream/2=. The wrapper proxies to =ReqLLM=, but taps the
=stream_response.stream= via =Stream.transform/4= to forward =:meta= chunks
into the caller process mailbox. =respond= drains usage at end of streaming.

* Tests
- Unit: =Concept.LLM.UsageTap= aggregates =:meta= chunks across iterations.
- Unit: respond change merges usage into =build_audit_attrs/2= output.
- Integration: full =:respond= action with a stubbed =req_llm= module verifying
  =prompt_tokens=, =completion_tokens= persist on the response message.

  =prompt_tokens=, =completion_tokens=, =latency_ms=.
