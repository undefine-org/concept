* Intention & Purpose

A bare keystroke must never crash a LiveView. Cmd-J (chat toggle) has no meaning on the landing page, so it should be a safe no-op — exactly as BUG-025 made Cmd-K safe there.

* Symptom (confirmed live, 2026-05-29)

=HomeLive= mounts =phx-hook="GlobalKeys"= on =#home-root=. =assets/js/hooks/global_keys.js= pushes =toggle_chat= on Cmd/Ctrl-J (and =open_command_palette= on Cmd-K). BUG-025 added =open_command_palette= + =close_command_palette= handlers to HomeLive, but **no =toggle_chat= clause exists**. Pressing Cmd-J on =/= (signed-out, or the brief pre-redirect window for signed-in) raises =Phoenix.LiveView.handle_event/3= no-clause → LV crash + reconnect.

* Root cause

GlobalKeys is a shared hook that emits three events (open_command_palette, toggle_chat, close_command_palette). Any LiveView mounting it must handle all three. HomeLive covers two. This is the same class as BUG-025 — a shared-hook contract gap.

NB structural: the hook's event set is an implicit contract. Consider a follow-up to make GlobalKeys' chat binding opt-in (e.g. =data-global-keys="palette"= vs ="palette chat"=) so a host that has no chat doesn't receive toggle_chat at all. Out of scope for this rescue; file FUP if pursued.

- OUT: rendering a chat panel on landing; reworking the hook contract (FUP).
