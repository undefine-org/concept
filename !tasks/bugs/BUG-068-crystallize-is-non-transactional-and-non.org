* Problem
CloneBlocks clones blocks one-by-one (Enum.map + Ash.create each, + create_link
each), then marks the conversation crystallized — NO surrounding transaction, no
undo/compensation. If a mid-loop clone fails, the {:ok, nb} = match raises →
rescue returns {:error}, but already-created page blocks + Links persist and the
conversation is left UNMARKED. Re-invoking clones the whole conversation again
(no guard on crystallized_page_id; Link unique_triple won't dedupe because each
run yields a new target_block_id). Net: half-crystallized pages on failure,
duplicate clones/links on retry.

* Fix
(1) Wrap the clone loop in Concept.Repo.transaction so partial failure rolls
back. (2) Idempotency guard: if conversation.crystallized_page_id already set,
no-op (or return existing). Add tests: failure mid-clone leaves page unchanged;
double-crystallize is a no-op.

* Acceptance
Crystallize is all-or-nothing; calling it twice does not duplicate blocks/links.
