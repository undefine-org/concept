* Intention & Purpose

~BlockChunker~ approximates tokens as ~div(byte_size(t), 4)~ and recomputes it
independently in ~coalesce_small/2~. For multibyte/CJK text this mis-counts,
skewing ~@target_tokens 512~ / ~@merge_below 128~ coalescing and reported
~embed_tokens~.

* Files
- ~lib/concept/knowledge/block_chunker.ex~

* Implementation
- Compute ~token_count~ once in ~render_block/1~, carry it on the chunk map, sum
  carried values in ~coalesce_small/2~ instead of recomputing from ~byte_size~.

* Acceptance
- ~coalesce_small/2~ no longer calls ~byte_size~; merged token_count == sum.
- Unit test: known token_counts coalesce deterministically.
