* Initial Message

Frontend audit (2026-05-16) flagged that =Layouts.app= advertises an optional ~current_scope~ attr (per Phoenix 1.8 / AGENTS.md mandate) but no LiveView passes one. AGENTS.md is explicit:

#+begin_quote
Always begin your LiveView templates with =<Layouts.app flash={@flash} ...>= which wraps all inner content
...
Anytime you run into errors with no ~current_scope~ assign: you failed to follow the Authenticated Routes guidelines
#+end_quote

Today this is latent: no compile-time crash because the attr is optional, no runtime crash because nothing inside =Layouts.app= dereferences it, but the moment any component starts using ~@current_scope~ — e.g. AshAdmin link visibility, Ash policies authorizing by scope, conditional sidebar items — we get the "no current_scope assign" error class AGENTS.md warns about. We also have the inverse failure mode (PLAN-002 BUG-003): WorkspaceLive uses bespoke assigns (=@current_user=, =@workspace=) that drift from scope.

* Intention & Purpose

Establish a single ~Concept.Accounts.Scope~ struct, assigned by an =on_mount= hook, threaded into every LiveView mount + every =Layouts.app= invocation. Replaces ad-hoc passing of (current_user, workspace, membership_role). Future Ash policies and live_components can rely on ~@current_scope~ existing for authenticated routes.

* Context & Narrative

Existing wiring:
- =lib/concept_web/live_user_auth.ex= — on_mount hooks ~live_user_required~, ~live_user_optional~, ~live_no_user~. Each assigns ~:current_user~.
- =lib/concept_web/components/layouts.ex= — =app/1= component receives ~flash~ + (implicitly) ~current_scope~. The slot is rendered but ~current_scope~ unreferenced.

#+begin_src elixir
defmodule Concept.Accounts.Scope do
  @moduledoc "Authenticated request/LV scope: user + optional workspace + role."
  defstruct [:user, :workspace, :role, system?: false]
  @type t :: %__MODULE__{user: Concept.Accounts.User.t(), workspace: Concept.Accounts.Workspace.t() | nil, role: :owner | :admin | :member | nil, system?: boolean()}

  def for_user(user, workspace_id \\ nil)
  def for_user(nil, _), do: nil
  def for_user(user, nil), do: %__MODULE__{user: user}
  def for_user(user, workspace_id) do
    case Concept.Accounts.get_membership(user.id, workspace_id, actor: user) do
      {:ok, %{workspace: ws, role: role}} -> %__MODULE__{user: user, workspace: ws, role: role}
      _ -> %__MODULE__{user: user}
    end
  end
end
