* Root causes (confirmed via codebase review)

** §1.1 Layouts.app wraps WorkspaceLive in max-w-2xl
  lib/concept_web/components/layouts.ex renders <main class="px-4 py-20"><div class="mx-auto max-w-2xl">. WorkspaceLive's flex h-screen workspace shell renders inside it. After sidebar 240px, canvas ~432px, with 96+96 padding twice (canvas + page-header), title content area collapses to ~48px → letter-per-line wrap.

** §1.2 ora-slash-menu always rendered visible
  assets/js/components/ora-slash-menu.js :host has `display: block; position: absolute` with no `:host(:not([visible])) { display: none }` rule. The `visible` attribute toggles nothing. Custom properties --menu-top/--menu-left are written but :host never consumes them.

** §1.3 --color-notion-hover referenced but undefined
  7+ usages of bg-notion-hover / var(--color-notion-hover) across command_palette_live.ex, composite_picker.ex, live_citation_rail.ex, app.css :: theme only has --color-notion-sidebar-hover. All hover highlights invisible.

** §1.4 app.css structural breakage
  - 4 .ora-citation-* rules placed BEFORE @import directives (spec violation)
  - Stray closing brace inside `.block-highlight` rule terminates @layer components mid-block; rules after it silently dropped
  - 3 separate `@layer components` blocks, fragile

** §1.5 Flash uses daisyUI (not loaded)
  core_components.ex flash/1 uses `toast toast-top toast-end alert alert-info` — daisyUI plugin is not in @plugin list. Flash flows inline at top-left of body.

** §1.6 Title Enter blurs without advancing
  content_editable.js: Enter blurs only. Notion behaviour is to focus next block (create if empty). Currently the caret disappears.

** §2 daisyUI half-migration
  daisyUI vendor exists at assets/vendor/daisyui.js but NOT loaded. Code still uses navbar/btn/btn-ghost/dropdown/avatar/menu/alert/toast/chat-bubble/loading-dots/bg-base-*/input input-primary/select select-bordered/table table-zebra in:
  - layouts.ex (navbar)
  - core_components.ex (flash, input, button, select, textarea, header, table, list)
  - chat_component.ex (whole component)
  - workspace_live/chat_panel.ex (header + scope/profile selectors)
  - indexing_pill.ex (popover)
  - router.ex (AshAuthentication.Phoenix.Overrides.DaisyUI)

** §3 Misc
  - indexing_pill.ex: duplicate `class=` attribute on `<li>` (silently dropped)
  - command_palette_live.ex: duplicate `phx-value-index` per item
  - cover_band.ex: `{@rest}` only spread on default branch — clicking colored cover dead
  - sidebar mx-2 + padding 8px double-offsets rows
  - page_tree padding-left inline overrides ora-sidebar-row padding (only left)
  - heroicon names referenced without size suffix or non-existent — leaves icon voids
  - Google Fonts fetched at runtime (privacy + blocking render)
