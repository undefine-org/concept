* Intention & Purpose

Wire the toolbar end-to-end with Lexical commands. Clicking B/I/U/S/code toggles the corresponding format on the current selection; clicking link opens =<ora-link-editor>= positioned near the selection, and "Apply" sets a link node on the selection. Selection bubble shows/hides based on Lexical selection range (collapsed vs. non-collapsed).

* Context & Narrative

Lexical 0.17 format commands (=@lexical/rich-text= + core):
- ~FORMAT_TEXT_COMMAND~ with payload ~"bold" | "italic" | "underline" | "strikethrough" | "code"~ — dispatch via ~editor.dispatchCommand(FORMAT_TEXT_COMMAND, "bold")~.
- Link: ~TOGGLE_LINK_COMMAND~ from =@lexical/link= with URL string or null.

=assets/js/lexical/commands.js= already exists (24 lines per audit) — extend with ~toggleFormat(editor, format)~ and ~setLink(editor, url)~ wrappers. Keep ora-block component as the editor owner; the toolbar should be a sibling that resolves "currently focused block" via document.activeElement or an explicit setter.

Selection-bubble positioning: =<ora-format-toolbar>= uses absolute positioning relative to the page-editor container. Listen for ~selectionchange~ on the document (filtered to selections inside one of our ora-block hosts) → compute caret rect via =window.getSelection().getRangeAt(0).getBoundingClientRect()= → set ~style.top/left~ + ~hidden~ attribute.

Link editor: dispatch ~request-link~ → page-level listener mounts =<ora-link-editor>= as overlay with anchor rect → user types URL → editor emits ~apply-link~ → call ~setLink(editor, url)~ → close overlay.
